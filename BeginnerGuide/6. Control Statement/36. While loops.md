## Do...while 루프

루프 `do...while`가 먼저 실행된 후, 프로그램은 조건을 검사합니다. 조건이 이면 `true`, 조건이 이면 루프가 반복됩니다 `false`. `do...while`루프는 실행 후에 조건을 검사하기 때문에 **사후 테스트 루프**`while` 라고도 합니다. 실행 전에 조건을 검사하는 루프 와 달리 , `do..while`루프는 종료 조건 루프입니다. 따라서 본문은 항상 최소 한 번은 실행됩니다.

`do`이 루프는 키워드, 본문, 그리고 `while(condition)`:의 세 부분으로 구성됩니다.

```kotlin
do {
    // body: do something
} while (condition)
```

아래 프로그램은 표준 입력에서 정수를 읽어서 표시합니다. 사용자가 를 입력하면 `0`프로그램은 해당 숫자를 출력하고 멈춥니다. 아래 예제는 루프의 동작을 보여줍니다.

```kotlin
fun main() {
    do {
        val n = readln().toInt()
        println(n)
    } while (n > 0)
}
```

## 결론

`while`이 주제에서는 두 가지 기본적이지만 매우 유용한 루프인 과 를 살펴보았습니다 `do...while`. 두 루프 모두 본문과 조건을 가지며, 실행 순서만 다릅니다. `while`루프는 본문 실행 전에 조건을 확인하고, `do...while`루프는 첫 번째 실행 후에 조건을 확인합니다.

실제로 프로그래머들은 . `do..while`만큼 자주 사용하지 않습니다 `while`. 이 루프를 사용하는 좋은 예로는 사용자가 특정 숫자나 문자열을 입력할 때까지 표준 입력에서 데이터를 읽는 프로그램이 있습니다. 이제 이러한 루프의 기본을 알게 되었습니다. 자, 연습해 봅시다!
