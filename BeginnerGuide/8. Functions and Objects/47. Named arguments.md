매개변수가 있는 함수를 호출할 때, 매개변수의 이름으로 인수를 전달할 수 있습니다. 이 방법은 특히 함수에 매개변수가 많은 경우 코드의 가독성을 향상시킬 수 있습니다. 또한, 함수 호출 시 전달되는 인수의 순서를 변경하는 데에도 편리합니다.

## 코드 가독성 향상

영화 티켓을 판매하는 계산원이 있다고 가정해 보겠습니다. 티켓 가격은 하루 종일 변동이 없습니다. 하루가 끝날 때 계산원이 얼마나 남는지 계산해야 합니다. 세 개의 매개변수를 갖는 함수를 사용하여 계산할 수 있습니다.

```kotlin
fun calcEndDayAmount(startAmount: Int, ticketPrice: Int, soldTickets: Int) =
        startAmount + ticketPrice * soldTickets
```

- `startAmount`현금 창구에 있는 초기 금액입니다.
    
- `ticketPrice`티켓의 가격입니다.
    
- `soldTickets`이날 판매된 티켓 수입니다.
    

이는 일반 함수이므로 다음과 같이 호출할 수 있습니다.

```kotlin
val amount = calcEndDayAmount(1000, 10, 500)  // 6000
```

이렇게 하면 잘 작동하지만, 한 가지 문제가 있습니다. 바로 인수가 불분명하다는 점입니다. 물론 표준 변수를 선언하고 전달할 수도 있지만, 때로는 변수가 아닌 리터럴을 사용해야 할 때가 있습니다. **[명명된 인수를](https://hyperskill.org/learn/step/4640 "코틀린에서 명명된 인수는 매개변수의 위치가 아닌 이름을 지정하여 함수에 인수를 전달하는 방법입니다. | 이를 통해 코드 가독성을 향상시키고 인수 순서를 변경할 수 있으며, 특히 일부 매개변수에 기본값이 있는 경우 더욱 그렇습니다. 명명된 인수는 오버라이드된 함수를 포함한 함수 호출에서 인수 값 앞에 인수 이름을 지정하여 사용할 수 있습니다. 이는 파생 클래스에서 함수를 재정의할 때 인수 이름을 유지할 수 있으므로 유용할 수 있습니다. 하지만 불필요한 복잡성을 피하고 코드의 명확성을 유지하려면 명명된 인수를 현명하게 사용하는 것이 중요합니다.")** 사용하면 코드의 가독성이 향상됩니다. 이 문제를 해결하려면 이 함수의 각 인수에 이름을 지정할 수 있습니다.

```kotlin
val amount = calcEndDayAmount(
    startAmount = 1000,
    ticketPrice = 10,
    soldTickets = 500
)  // 6000
```

이제 코드를 이해하기 쉬워졌습니다!

## 인수 재정렬

명명된 인수를 사용하면 함수 호출에서 인수 순서를 변경할 수 있습니다. 원하는 순서대로 이름을 지정하기만 하면 됩니다.

```kotlin
val amount = calcEndDayAmount(
    ticketPrice = 10,
    soldTickets = 500,
    startAmount = 1000
)  // 6000
```

## 명명된 인수와 위치 인수

**명명된 인수와 일반** (위치) 인수를 사용하여 함수를 호출할 수도 있습니다 . 명명된 인수가 위치 인수 뒤에 배치되어야 합니다.

```kotlin
calcEndDayAmount(1000, ticketPrice = 10, soldTickets = 500)  // 6000
```

Kotlin 1.4 이상에서는 명명된 인수 뒤에 [위치 인수를](https://hyperskill.org/learn/step/4640 "코틀린에서 위치 인수는 특정 이름이 아닌 함수 호출 내 위치로 식별되는 함수 인수 유형입니다. | 즉, 위치 인수는 함수 시그니처에 정의된 순서대로 함수에 전달되어야 합니다. 코틀린 1.4 이전에는 위치 인수를 함수 호출에서 명명된 인수 앞에 배치해야 했습니다. 그러나 코틀린 1.4부터는 위치 인수가 원래 순서를 유지하는 한 명명된 인수 뒤에도 배치될 수 있습니다.") 사용할 수 있지만 , 인수 순서를 유지해야 합니다. 예:

```kotlin
calcEndDayAmount(ticketPrice = 10, 500, 1000)   // Incorrect invocation!
calcEndDayAmount(startAmount = 1000, 10, 500)   // OK


calcEndDayAmount(soldTickets = 500, ticketPrice = 10, startAmount = 500) // OK
calcEndDayAmount(soldTickets = 500, ticketPrice = 10, 500)  // Incorrect invocation!
```

## 기본 및 명명된 인수

[명명된 인수와 기본 인수를](https://hyperskill.org/learn/step/4640 "코틀린에서 기본 인수는 함수 호출 시 인수가 전달되지 않으면 기본값을 받는 함수의 매개변수입니다. | 기본 인수를 사용하면 기본값을 가진 모든 매개변수에 인수를 제공할 필요가 없으므로 함수 호출을 더 간결하게 만들 수 있습니다. 그러나 기본 인수와 일반 인수를 함께 사용할 경우 어떤 매개변수에 인수를 받아야 할지 모호할 수 있습니다. 혼란을 피하기 위해 명명된 인수를 기본 인수와 함께 사용할 수 있습니다. 기본값은 상수이거나 명명된 인수 또는 함수일 수 있지만, 반드시 초기화해야 합니다.") 함께 사용할 수도 있습니다 . 아시다시피 기본 인수는 Kotlin이 어떤 매개변수를 할당해야 하는지 이해하지 못하기 때문에 혼란스러울 수 있습니다.

이전 함수를 수정하여 첫 번째 매개변수를 선택 사항으로 만들어 보겠습니다.

```kotlin
fun calcEndDayAmount(startAmount: Int = 0, ticketPrice: Int, soldTickets: Int) =
        startAmount + ticketPrice * soldTickets
```

마지막 두 개의 매개변수만 사용하여 이 함수를 호출하려고 한다고 가정해 보겠습니다. 하지만 작동하지 않을 것입니다.

```kotlin
val amount = calcEndDayAmount(10, 500)  // Won't work —
                                        // no value for soldTickets
```

여기서는 두 번째 매개변수가 아닌 `10`첫 번째 선택적 인수에 할당됩니다 .`startAmount``ticketPrice`

이 함수를 호출하려면 명명된 인수를 사용하세요.

```kotlin
val amount = calcEndDayAmount(ticketPrice = 10, soldTickets = 500)  // 5000
```

이제 작동합니다!

## 명명된 인수 및 기본값

함수 매개변수의 기본값은 상수일 수도 있고 상수가 아닐 수도 있습니다. 또한, 다른 명명된 인수나 함수일 수도 있습니다.

예를 들어:

```kotlin
fun sum2(a: Int, b: Int = a) = a + b
 
sum2(1)    // 1 + 1
sum2(2, 3) // 2 + 3
```

다음 코드는 `b`매개변수가 초기화되지 않았으므로 작동하지 않습니다.

```kotlin
fun sum2(a: Int = b, b: Int) = a + b
```

## 결론

요약하자면, **명명된 인수를** 현명하게 사용하여 코드 가독성을 높이고 일부 매개변수에 기본값이 있는 경우 인수 순서를 변경하세요.
