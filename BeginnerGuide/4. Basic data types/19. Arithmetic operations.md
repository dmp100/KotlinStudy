우리는 일상생활에서 종종 [산술 연산을](https://hyperskill.org/learn/step/4472 "Kotlin에서 산술 연산은 특정 연산자를 사용하여 숫자 값에 대해 수행되는 수학적 계산입니다. Kotlin에는 5가지 산술 연산자가 있습니다. 더하기(+), 빼기(-), 곱하기(\*), 나누기(/), 모듈로(%). 이러한 연산자는 이진 연산자로, 연산자가 적용되는 값이나 변수인 두 개의 피연산자를 사용합니다. 예를 들어, 표현식 1 + 3에서 1과 3은 피연산자이고 +는 연산자입니다. Kotlin의 산술 연산은 곱셈이 덧셈이나 뺄셈보다 우선순위가 높은 것과 같은 특정 규칙을 따릅니다. 괄호를 사용하여 실행 순서를 지정하고 표기법을 단순화할 수 있습니다. 또한 Kotlin은 산술 연산과 할당을 결합하는 복합 할당 연산을 지원합니다. 예를 들어 덧셈 후 할당(a += b) 및 뺄셈 후 할당(a -= b)이 있습니다.") 합니다 . 변화를 계산하고, 방의 넓이를 측정하고, 줄을 서 있는 사람 수를 세는 등 여러 가지 작업을 해야 합니다. 컴퓨터는 이러한 작업을 도와줍니다.

## 이진 연산자

Kotlin에는 5개의 [산술 연산자가](https://hyperskill.org/learn/step/4472 "Kotlin에서 산술 연산자는 덧셈, 뺄셈, 곱셈, 나눗셈, 모듈로 나눗셈과 같은 수학 연산을 수행하는 데 사용되는 기호입니다. | 이러한 연산자는 피연산자라고도 하는 두 개의 값 또는 변수를 입력으로 사용하여 단일 출력을 생성합니다. 예를 들어, 1 + 3 표현식에서 1과 3은 피연산자이고 +는 연산자입니다. 곱셈은 덧셈이나 뺄셈보다 우선순위가 높으므로 먼저 계산됩니다. 괄호를 사용하여 실행 순서를 지정하고 표기법을 단순화할 수 있습니다. 예를 들어, 아래 프로그램은 산술 연산을 사용하여 숫자를 역순으로 출력하여 숫자를 추출합니다. Kotlin에서 나누기 연산자는 슬래시(/)로 표시합니다. 두 숫자를 나누고 몫을 반환합니다.") 있습니다 .

- 덧셈`+`
- 빼기`-`
- 곱셈`*`
- 정수 나누기`/`
- 계수`%`

이러한 연산자는 이진 연산자입니다. 즉, 두 개의 값을 [피연산자](https://hyperskill.org/learn/step/4472 "코틀린에서 피연산자는 연산자가 적용되는 값 또는 변수입니다. | 예를 들어, 표현식 1 + 3에서 1과 3은 피연산자이고 +는 연산자입니다. 피연산자는 리터럴, 변수 또는 더 복잡한 표현식일 수 있습니다. 코틀린에는 덧셈(+), 뺄셈(-), 곱셈(*), 나눗셈(/), 나머지(%)의 다섯 가지 산술 연산자가 있습니다. 이러한 연산자는 이진 연산자로, 두 개의 피연산자를 사용합니다. 피연산자의 평가는 연산자 우선순위라는 일련의 규칙에 따라 수행되며, 연산자는 이 규칙에 따라 적용됩니다.") 로 사용합니다 . 피연산자는 연산자가 적용되는 값 또는 변수입니다. 예를 들어, 표현식 1 + 3에서 1과 3은 피연산자이고, +는 연산자입니다.

아래 코드는 **덧셈** , **뺄셈** , **곱셈** 의 결과를 출력합니다 .

```kotlin
println(13 + 25) // prints 38
println(20 + 70) // prints 90

println(70 - 30) // prints 40
println(30 - 70) // prints -40

println(21 * 3)  // prints 63
println(20 * 10) // prints 200
```

이 `/`연산자는 두 수의 정수 부분을 **나누고** , 소수 부분은 버립니다. 모듈로 나눗셈에 대한 내용은 에서 확인하실 수 [있습니다](https://hyperskill.org/learn/step/10586) .

```kotlin
println(8 / 3)  // prints 2
println(41 / 5) // prints 8
```

연산자 는 나누기의 **나머지를**`%` 구합니다 .

```kotlin
println(10 % 3) // prints 1 because 10 divided by 3 leaves a remainder of 1
println(12 % 4) // prints 0 because 12 divided by 4 leaves no remainder
```

아시다시피 음수 나눗셈은 다른 계산 방식입니다. 다양한 나눗셈 유형에 대해 자세히 알아보려면 이 [글을](https://hyperskill.org/learn/step/22102) 읽어보세요 .

## 복잡한 표현

[산술 연산을 결합하여 복잡한 표현식을](https://hyperskill.org/learn/step/4472 "코틀린에서 복합 표현식은 단일 값을 생성하는 코드를 의미하지만, 더 복잡하며 여러 연산, 함수 또는 람다 표현식을 포함할 수 있습니다. | 복합 표현식은 수학적 표현식, 논리적 표현식 또는 이 둘의 조합일 수 있습니다. 복합 표현식은 특히 연산자 오버로딩, 람다 표현식, 한정된 반환 구문과 같은 코틀린의 강력한 기능과 함께 사용할 때 코드를 더 간결하고 읽기 쉽게 만들 수 있습니다. 하지만 이러한 표현식을 신중하게 사용하고 코드의 명확성과 유지 관리 용이성을 저해하지 않도록 하는 것이 중요합니다.") 작성할 수 있습니다 .

```kotlin
println(1 + 3 * 4 - 2) // prints 11
```

계산 순서는 산술 연산 규칙과 일치합니다. 곱셈은 덧셈이나 뺄셈보다 우선순위가 높으므로 `3 * 4`먼저 계산됩니다.

값을 생성하는 코드의 일부로 표현식을 생각해 보세요(예: `14 + 5`또는 `15 - 3 * 4`). 더 나아가 표현식을 자세히 살펴보겠습니다.

실행 순서를 지정하려면 **괄호를** 사용하세요 . 아래 예를 살펴보겠습니다.

```kotlin
println((1 + 3) * (4 - 2)) // prints 8
```

괄호는 중첩될 수 있습니다. 또한, 표기법을 단순화하기 위해 괄호를 사용할 수도 있습니다.

예를 들어, 다음은 숫자의 모든 자릿수를 `54`역순으로 출력하는 프로그램입니다. 산술 연산을 사용하여 자릿수를 추출합니다.

```kotlin
fun main() {
    println(54 % 10) // it prints 4
    println((54 / 10) % 10) // it prints 5
}
```

프로그램 출력:

```no-highlight
4
5
```

## 단항 연산자

[단항 연산자는](https://hyperskill.org/learn/step/4472 "Kotlin에서 단항 연산자는 하나의 피연산자 또는 값에서만 작동하는 연산자 유형입니다.| 두 개의 피연산자에서 작동하는 이항 연산자와 다릅니다.단항 연산자는 피연산자 앞에 배치되는 단항 접두사 연산자(예: 증가 및 감소 연산자)이거나 피연산자 뒤에 배치되는 단항 접미사 연산자입니다.Kotlin에서 단항 연산자를 사용자 지정하려면 해당 유형에 대한 특정 이름을 가진 멤버 함수 또는 확장 함수를 제공해야 합니다.이 유형은 이항 연산의 왼쪽 유형이 되고 단항 연산의 인수 유형이 됩니다.함수를 연산자로 표시하려면 `operator` 수정자를 사용해야 합니다.단항 접두사 연산자 표현식은 단일 매개 변수가 있는 함수 호출로 변환되는 반면, 증가 및 감소 표현식은 매개 변수가 없는 함수 호출로 변환된다는 점에 유의하는 것이 중요합니다.단항 연산자는 Kotlin에서 연산자 오버로딩의 일부로, 특정 유형에 대해 연산자의 작동 방식을 변경할 수 있습니다.") 피연산자 로 단일 값을 취합니다.

- 단항 **더하기 연산자** 는 값만 제공합니다. 선택 연산자이므로 실제로는 생략합니다.

```kotlin
println(+5) // prints 5
println(+(-5)) // prints -5
```

- 단항 **빼기는** 값이나 표현식을 부정합니다.

```kotlin
println(-8)  // prints -8
println(-(100 + 4)) // prints -104
```

두 연산자 모두 곱셈 및 나눗셈 연산자보다 **[우선순위가 높습니다 .](https://hyperskill.org/learn/step/4472 "코틀린에서 우선순위는 표현식에서 연산의 순서를 의미하며, 어떤 연산을 다른 연산보다 먼저 수행해야 하는지를 지정하는 규칙에 따라 결정됩니다. | 우선순위가 높은 연산이 먼저 수행되고, 그 다음에 우선순위가 낮은 연산이 수행됩니다. 예를 들어, 곱셈은 덧셈보다 우선순위가 높으므로 2 + 3 * 4와 같은 표현식에서 곱셈이 먼저 수행되어 14 + 3 = 17이 됩니다. 괄호를 사용하면 연산의 우선순위를 높이거나 코드를 더 명확하게 표현할 수 있습니다.")**

## 우선순위

[괄호를 포함한 산술 연산자의 우선순위를](https://hyperskill.org/learn/step/4472 "Kotlin에서 우선순위는 표현식에서 연산을 수행하고 그룹화하는 순서를 결정하는 일련의 규칙입니다. 우선순위가 높은 연산은 우선순위가 낮은 연산보다 먼저 수행됩니다. Kotlin에서 우선순위에 대한 몇 가지 주요 사항은 다음과 같습니다. * 우선순위는 표현식에서 연산의 순서를 결정합니다. 예를 들어 곱셈은 덧셈보다 우선순위가 높으므로 2 + 3 \* 4와 같은 표현식이 있는 경우 곱셈이 먼저 수행되어 14 + 3 = 17이 됩니다. * 괄호는 연산의 우선순위를 높이거나 코드를 더 명확하게 만드는 데 사용할 수 있습니다. * 연산자의 우선순위가 같은 경우 실행 순서를 결정하는 데 다른 규칙이 사용됩니다. 우선순위가 같은 연산자의 기본 규칙은 왼쪽에서 오른쪽입니다. * 올바른 계산을 위해 우선순위를 명심하는 것이 중요합니다. 순서를 기억하기 위한 일반적인 약어는 BODMAS(Bracket, Order, Division, Multiplication, Addition, Subtraction)입니다. * 증가 및 감소와 같은 단항 연산자는 곱셈 및 나눗셈 연산자보다 우선순위가 높습니다.") 살펴보세요 . 목록은 가장 높은 우선순위부터 가장 낮은 우선순위 순으로 정렬되어 있습니다.

1. 괄호;
2. 단항 플러스/마이너스;
3. 곱셈, 나눗셈, 모듈러스
4. 덧셈과 뺄셈.

이 순서를 기억하세요. 정확한 계산을 위해 중요합니다. 또한, BODMAS(괄호, 순서, 나눗셈, 곱셈, 덧셈, 뺄셈)라는 약어를 사용하여 우선순위를 기억할 수 있습니다.

## 결론

`+`이 주제에서는 다섯 가지 주요 산술 연산( , `-`, `*`, `/`, ) 을 다루었습니다 `%`. 물론, 괄호(표기법을 단순화하기 위해)와 단항 연산자를 사용하여 이러한 연산을 결합하여 복잡한 표현식을 만들 수 있습니다. 하지만 우선순위에 유의해야 합니다. 다행히 코틀린에서는 우선순위가 고등학교에서 배운 규칙을 따릅니다.
