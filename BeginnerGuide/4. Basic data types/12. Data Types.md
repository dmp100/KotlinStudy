## 데이터 유형이란 무엇인가요?

숫자와 텍스트는 상당히 다르다는 것을 우리 모두 알고 있습니다. 어떻게 알 수 있을까요? 숫자에는 [산술 연산](https://hyperskill.org/learn/step/4388 "Kotlin에서 산술 연산은 특정 연산자를 사용하여 숫자 값에 대해 수행되는 수학적 계산입니다. Kotlin에는 5가지 산술 연산자가 있습니다. 더하기(+), 빼기(-), 곱하기(\*), 나누기(/), 모듈로(%). 이러한 연산자는 이진 연산자로, 연산자가 적용되는 값이나 변수인 두 개의 피연산자를 사용합니다. 예를 들어, 표현식 1 + 3에서 1과 3은 피연산자이고 +는 연산자입니다. Kotlin의 산술 연산은 곱셈이 덧셈이나 뺄셈보다 우선순위가 높은 것과 같은 특정 규칙을 따릅니다. 괄호를 사용하여 실행 순서를 지정하고 표기법을 단순화할 수 있습니다. 또한 Kotlin은 산술 연산과 할당을 결합하는 복합 할당 연산을 지원합니다. 예를 들어 덧셈 후 할당(a += b) 및 뺄셈 후 할당(a -= b)이 있습니다.") (예: 곱셈)을 수행할 수 있지만 텍스트에는 수행할 수 없습니다. 코틀린도 이를 알고 있습니다. 그렇기 때문에 모든 변수는 해당 변수에 수행할 수 있는 연산과 저장할 수 있는 값을 결정하는 **타입을 가지고 있습니다.**

## 변수 유형

변수의 유형은 변수가 선언될 때 설정됩니다.

```kotlin
val text = "Hello, I am studying Kotlin now."
val n = 1
```

이 경우, 코틀린은 `text`가 문자열이고 `n`가 숫자라는 것을 알고 있습니다. 코틀린은 두 변수의 타입을 자동으로 판단합니다. 이 메커니즘을 **[타입 추론](https://hyperskill.org/learn/step/4388 "Kotlin에서 타입 추론은 컴파일러가 명시적인 타입 선언 없이 변수, 표현식 또는 함수 매개변수의 데이터 타입을 자동으로 추론하는 프로세스입니다. | 이 기능은 특히 람다 표현식에서 매개변수 타입이 인터페이스에서 추론되어 코드를 더 쉽게 읽을 수 있도록 하는 데 유용합니다. Kotlin의 타입 추론은 Java의 함수형 인터페이스와 완벽하게 호환되어 두 인터페이스 간의 원활한 상호 운용성을 제공합니다. 타입 추론은 프로그래머가 매번 변수의 타입을 명시적으로 지정할 필요가 없으므로 코드의 중복을 줄이고 코드를 더 간결하게 만드는 데 도움이 됩니다.")** 이라고 합니다 .

다음 코드를 살펴보세요. 타입 추론을 사용하여 변수를 선언하는 방법은 다음과 같습니다.

```kotlin
val/var identifier = initialization
```

변수를 선언할 때 변수의 유형을 지정할 수도 있습니다.

```kotlin
val/var identifier: Type = initialization 
```

참고로, 유형 이름은 항상 대문자로 시작합니다.

이전 예제와 동일한 변수를 선언하고 해당 유형을 지정해 보겠습니다.

```kotlin
val text: String = "Hello, I am studying Kotlin now."
val n: Int = 1
```

이 `Int`유형은 변수가 정수( `0`, `1`, `2`, ..., `100_000_000`, ...)를 저장한다는 것을 의미합니다. 이 `String`유형은 변수가 문자열( `"Hello"`, )을 저장한다는 것을 의미합니다. 나중에 이러한 [데이터 유형](https://hyperskill.org/learn/step/4388 "In Kotlin, a data type is a classification that specifies the kind of value that a variable, expression, or function parameter can hold. | It is an essential aspect of a statically-typed language like Kotlin, which requires explicit or implicit type declarations for variables and function parameters. For instance, Kotlin has various data types, such as integers, floating-point numbers, characters, booleans, and strings. Among these, strings are a widely-used data type that represents a sequence of zero or more characters enclosed by double quotes. Strings are often used in conditional statements and regular expressions, and they have various operations, such as getting the length or concatenating two strings. Moreover, type inference is a helpful feature in Kotlin that allows the compiler to deduce the data type of a variable, expression, or function parameter automatically, making the code more concise and readable.")`"John Smith"` 과 다른 데이터 유형에 대해 자세히 알아보겠습니다 .[](https://hyperskill.org/learn/step/4388 "Kotlin에서 데이터 유형은 변수, 표현식 또는 함수 매개변수가 보유할 수 있는 값의 종류를 지정하는 분류입니다. | Kotlin과 같은 정적 타입 언어의 필수적인 측면으로, 변수와 함수 매개변수에 대한 명시적 또는 암시적 타입 선언이 필요합니다. 예를 들어 Kotlin에는 정수, 부동 소수점 숫자, 문자, 부울, 문자열과 같은 다양한 데이터 유형이 있습니다. 이 중 문자열은 큰따옴표로 묶인 0개 이상의 문자 시퀀스를 나타내는 널리 사용되는 데이터 유형입니다. 문자열은 조건문과 정규 표현식에서 자주 사용되며 길이를 얻거나 두 문자열을 연결하는 등 다양한 연산을 수행합니다. 또한, 타입 추론은 Kotlin의 유용한 기능으로, 컴파일러가 변수, 표현식 또는 함수 매개변수의 데이터 유형을 자동으로 추론하여 코드를 더 간결하고 읽기 쉽게 만듭니다.")

실제로 사람들이 이 두 가지 형태의 변수 선언을 모두 사용하는 것을 보게 될 것입니다. 타입 추론을 사용하면 코드가 더 간결하고 가독성이 높아지지만, 경우에 따라서는 타입을 명시하는 것이 더 나을 수 있습니다. 예를 들어, 변수를 선언하고 나중에 [초기화](https://hyperskill.org/learn/step/4388 "코틀린에서 초기화는 변수가 선언될 때 초기값을 할당하는 것을 의미합니다. `=` 기호 다음에 값을 지정하여 초기화할 수 있습니다. `val` 키워드를 사용하는 경우 초기화가 필수적입니다. 기본 클래스에 생성자 매개변수가 있는 경우 파생 클래스에서 이를 처리해야 합니다. 코틀린은 기본 클래스가 제대로 초기화되지 않으면 프로그램 컴파일을 허용하지 않습니다. 초기화는 간단한 객체의 경우 가벼운 작업이지만, 더 복잡한 클래스의 인스턴스를 생성하는 것은 비용이 많이 들 수 있습니다.") 해야 하는 경우 , 타입 추론은 전혀 작동하지 않습니다.

```kotlin
val greeting // error
greeting = "hello"
```

위 예제는 Kotlin이 변수가 선언만 되어서는 타입을 유추할 수 없기 때문에 올바르지 않습니다. 모든 변수는 타입을 가져야 합니다. 반대로, 아래 예제는 프로그래머가 타입을 지정하기 때문에 작동합니다.

```kotlin
val greeting: String // ok
greeting = "hello"
```

## 유형 불일치

데이터 유형의 가장 중요한 기능 중 하나는 변수에 부적절한 값을 할당하는 것을 방지하는 것입니다. 작동하지 않는 코드의 예를 살펴보겠습니다.

```kotlin
val n: Int = "abc" // Type mismatch: inferred type is String but Int was expected
```

**[따라서 타입 불일치](https://hyperskill.org/learn/step/4388 "코틀린에서 타입 불일치는 변수에 다른 타입의 값을 할당하거나, 타입 추론을 사용하여 선언된 변경 가능한 변수에 적합하지 않은 값을 할당하려고 할 때 발생하는 오류입니다. | 타입 불일치 오류 메시지에는 일반적으로 \"타입 불일치\"라는 메시지가 포함되며, 예상 타입과 실제 타입이 명시됩니다. 이 오류는 서로 다른 타입이 만나고 타입 강제 변환이 불가능할 때 발생합니다. 타입 강제 변환은 타입 계층 구조에서 더 넓은 타입으로 값을 자동 변환하는 것으로, 정보 손실 없이 값을 한 타입에서 다른 타입으로 변환합니다.")** 오류가 표시되면 변수에 적합하지 않은 값을 할당했다는 의미입니다. 타입 추론을 사용하여 선언된 변경 가능한 변수에 적합하지 않은 값을 할당하려고 할 때도 같은 문제가 발생합니다.

```kotlin
var age = 30 // the type is inferred as Int
age = "31 years old" // Type mismatch
```

변수의 유형은 변경할 수 없습니다.

## 결론

나중에 다른 데이터 유형과 그와 관련된 몇 가지 문제에 대해 알아보겠습니다. 이제부터 기억해야 할 점은 변수가 생성되면 유형이 정해지며, 나중에 유형을 변경할 수 없다는 것입니다.
