`readln()`때로는 표준 입력에서 데이터를 읽는 것만으로 는 충분하지 않습니다 . `readln()`이는 단순한 방법일 뿐, 복잡한 작업에는 적합하지 않습니다. 예를 들어, 데이터 흐름에서 특정 데이터만 찾고 `Integers`, `Doubles`데이터를 단어 단위로 순차적으로 읽고, 이러한 읽기 작업을 단계별로 제어해야 하는 경우입니다. 이 주제에서는 Kotlin에서도 사용할 수 있는 Java 도구 중 하나인 `Scanner`.

## Java Scanner란 무엇인가요?

Scanner는 표준 입력에서 데이터를 가져오는 또 다른 방법입니다. 다른 Java 라이브러리와 상호 운용되므로 Kotlin에서 직접 접근할 수 있습니다. Scanner를 사용 `Scanner`하면 프로그램이 표준 입력에서 **다양한 유형** (문자열, 숫자 등)의 값을 읽을 수 있습니다.

이를 사용하려면 소스 코드가 있는 파일의 맨 위에 다음 import 문을 추가해야 합니다.

```kotlin
import java.util.Scanner
```

또는

```kotlin
import java.util.*
```

두 방법 모두 프로그램에 추가할 수 있습니다 `Scanner`. 가져오기에 대해서는 나중에 더 자세히 설명하겠습니다.

다음으로 초기화된 변수를 만들어 보겠습니다 `Scanner`.

```kotlin
val scanner = Scanner(System.`in`)
```

이제 표준 입력에서 데이터를 읽을 수 있습니다.

```kotlin
val line = scanner.nextLine() // read a whole line, e.g., "Hello, Kotlin"
val num = scanner.nextInt()   // read a number, e.g., 123
val string = scanner.next()   // read a string, e.g., "Hello"
```

`scanner.next()`**한 줄이 아니라 한 단어** 만 읽습니다 . 사용자가 를 입력하면 `Hello, Kotlin`를 읽습니다 `Hello,`.

`scanner.nextLine()`또는 이와 유사한 것을 호출하면 `scanner.nextInt()`프로그램은 입력 데이터를 예상합니다. 다음은 올바른 입력 데이터의 예입니다.

```kotlin

```

아래의 입력 예시도 유효합니다.

```kotlin

```

숫자가 새 줄에 있는 경우 `scanner.next()`또는 를 사용하여 숫자를 문자열로 읽을 수 있습니다 .`scanner.nextLine()`

또한, 이 유형은 다른 유형의 값을 읽을 수 있는 여러 메서드(함수)를 제공합니다. 자세한 내용은 [클래스 스캐너](https://docs.oracle.com/javase/8/docs/api/java/util/Scanner.html)`Scanner` 문서를 참조하세요 .[](https://docs.oracle.com/javase/8/docs/api/java/util/Scanner.html)

아래 프로그램은 두 개의 숫자를 읽어 두 개의 다른 줄에 역순으로 출력합니다.

```kotlin
import java.util.Scanner // a class (type) from the Java standard library

fun main() {
    val scanner = Scanner(System.`in`) // reads data

    val num1 = scanner.nextInt() // reads the first number
    val num2 = scanner.nextInt() // reads the second number

    println(num2) // prints the second number
    println(num1) // prints the first number
}
```

좋아요, 이제 데이터를 읽었으니 `Scanner`더 이상 필요 없습니다. 를 사용했을 때는 `readln()`코드를 그대로 진행하면서 더 나아갔습니다. 하지만 `Scanner`PC 리소스는 계속 소모될 수 있습니다. 이를 막으려면 를 사용해야 하는데, `close()`이 메서드는... 취소 기능을 합니다 `Scanner`:)

```kotlin
scanner.close()
```

이제 괜찮습니다.

## 사용자 정의 구분 기호

중요한 점 하나: 데이터를 직접 입력할 수 있습니다 `Scanner`. 확인해 보겠습니다.

```kotlin
val scanner = Scanner("123_456")
```

하지만 공백이나 줄 바꿈이 없으면 다음 단어를 어떻게 읽을 수 있을까요? 이런 경우, 에는 `Scanner`편리한 도구인 . 가 있습니다 `useDelimiter()`. . 를 사용하면 `next()`, `nextInt()`, 등의 메서드에 대한 구분 기호를 사용자 지정할 수 있습니다. (구분 기호의 기본값은 공백입니다.)

이러한 단어와 정수를 따로 읽으려면 새로운 구분 기호를 설정해야 합니다 `"_"`.

```kotlin
scanner.useDelimiter("_")
```

이제 다음 데이터를 읽을 수 있습니다.

```kotlin
println(scanner.nextInt()) // 123
println(scanner.nextInt()) // 456
```

시원한!

## 다음 요소를 확인하세요

스캐너에 다음과 같은 데이터가 있다고 가정해 보겠습니다.

```kotlin
val scanner = Scanner("Hello, Kotlin!")
```

이제 여러 단어를 읽어보도록 하겠습니다.

```kotlin
val word1 = scanner.next()
val word2 = scanner.next()
val word3 = scanner.next()
```

세 번째 단어를 읽으려고 하면 예외가 발생하고 프로그램이 다운됩니다. 이럴 때 어떻게 해야 할까요? 허, `Scanner`이런 상황에 대비한 자체 메서드가 있습니다. 바로 `hasNext()`( `hasNextInt()`, 등)입니다.

```kotlin
if (scanner.hasNext()) val word1 = scanner.next() // Hello,
if (scanner.hasNext()) val word2 = scanner.next() // Kotlin!
if (scanner.hasNext()) val word3 = scanner.next() // It doesn't work but also doesn't make error
```

이제 데이터 흐름이 종료되면 데이터를 읽을 수도 있고 읽지 않을 수도 있습니다.

## 결론

이 주제에서는 표준 입력에서 데이터를 읽는 두 번째 방법인 Java에 대해 살펴보았습니다 `Scanner`. 이는 복잡한 데이터를 읽고 콘텐츠를 필터링하는 편리한 방법입니다. 다음 사항을 기억하세요.

1. 필요하지 않으면 `Scanner`항상 .을 사용하세요 `readln()`.
    
2. 다양한 유형의 데이터를 읽기 위해 Scanner에는 `next()`, `nextInt()`, 와 같은 메서드가 있습니다 `nextLine()`.
    
3. 이상한 구분 기호가 포함된 입력이 있는 경우 다음 방법을 사용하세요 `useDelimiter()`.
    
4. 입력 내용에 단어가 몇 개인지 모르는 경우에는 다음 방법을 사용하세요 `hasNext()`.
