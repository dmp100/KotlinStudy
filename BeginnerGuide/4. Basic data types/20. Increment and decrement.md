## 할당 작업

Kotlin이 덧셈, 뺄셈과 같은 기본적인 산술 연산을 지원한다는 사실은 이미 알고 계실 겁니다.

```kotlin
var a = 3
a = a + 1 // 4
a = a - 1 // 3
```

이 외에도 산술 연산과 대입 연산을 결합한 **복합** **[대입 연산이](https://hyperskill.org/learn/step/10776 "코틀린에서 할당 연산은 변수나 속성에 값을 할당하는 방법입니다. | 기호는 = 로 표시됩니다. 코틀린은 기본적인 할당 연산 외에도 산술 연산과 할당을 결합한 복합 할당 연산도 지원합니다. 예를 들어, `a += b`는 `a = a + b`와 같습니다. 이러한 연산은 변수 이름을 반복할 필요성을 없애 코드를 간결하고 읽기 쉽게 만듭니다. 덧셈, 뺄셈, 곱셈, 나눗셈, 모듈러스 할당에도 사용할 수 있습니다.")** 있습니다 . 대입 연산자는 `=`변수가 두 번 반복되는 것을 방지하기 위해 연산과 결합하는 여러 가지 형태를 가지고 있습니다.

- `+=`덧셈 후 할당: **A += B는** **A = A + B** 와 같음
    
- `-=`뺄셈 후 할당: **A -= B는** **A = A - B** 와 같습니다 .
    
- `*=`곱셈 후 할당: **A *= B는** **A = A * B** 와 같습니다 .
    
- `/=`나눗셈 후 할당: **A /= B는** **A = A / B** 와 같습니다 .
    
- `%=`나눗셈 후 나머지 할당: **A %= B** = **A = A % B**
    

단일 변수에 수행되는 할당 연산을 살펴보겠습니다. 아주 기본적인 내용은 다음과 같습니다.

```kotlin
var a = 3
a += 2 // 5
a -= 2 // 3
a *= 2 // 6
a /= 2 // 3
a %= 2 // 1
```

여기서는 숫자 2와 변수에 대한 계산을 수행 `a`하고 값을 할당했습니다. 보시다시피 이러한 연산 덕분에 코드가 더 짧고 명확해졌습니다.

복합 할당 연산자는 이미 정의된 변수에만 적용할 수 있으며 새 변수를 선언하는 데 사용할 수 없습니다.

```kotlin
var a: Int
a += 5 // compile-time error, Variable 'a' must be initialized
```

## 증가와 감소를 사용하세요

또 다른 일반적인 연산은 숫자를 1씩 증가시키거나 감소시키는 것입니다. 물론 `+= 1`or 를 사용할 수도 있지만 `-= 1`, 코틀린은 이보다 더 나은 방법인 **증가** 및 **감소** 연산을 제공합니다. 예를 들어 보겠습니다.

```kotlin
var num = 3
num++  // 4, increment
num--  // 3, decrement
```

위의 코드는 실제로 아래와 같습니다.

```kotlin
var num = 3
num += 1  // 4
num -= 1  // 3
```

보시다시피, increase 는 `++`increase 와 동일한 기능을 `+=1`하지만 더 간단한 방식으로 수행합니다. 감소 의 경우도 마찬가지입니다 `--`.

이 기능은 숫자를 1씩 늘리거나 줄일 때만 작동합니다.

보기에는 쉬워 보이지만, 증가 연산자와 감소 연산자는 생각보다 훨씬 복잡합니다. 증가 연산자와 감소 연산자는 모두 **접두사(prefix)** 와 접미사 **(postfix)라는** 두 가지 형태를 가지는데, 이 두 가지 형태를 구분하는 것이 매우 중요합니다 .

## 접두사 형태

접두사 형태는 변수가 사용되기 전에 값을 변경합니다. 몇 가지 예를 살펴보겠습니다.

접두사 증가는 증가된 값을 반환합니다.

```kotlin
var a = 10
val b = ++a
println(a)  // a = 11
println(b)  // b = 11
```

먼저 변수의 값을 `a`1 증가시킨 후, 그 값을 변수 에 할당합니다 `b`. 따라서 `a`와 는 `b`모두 11입니다.

접두사 감소는 다음을 반환합니다.

```kotlin
var a = 10
val b = --a
println(a)  // a = 9
println(b)  // b = 9
```

여기서도 같은 일이 일어나는 것을 볼 수 있습니다. 변수의 값이 `a`하나 감소한 후, 그 값이 변수에 할당됩니다 `b`.

## 접미사 형태

이와 대조적으로, 후위형은 변수가 사용된 후에 그 값을 변경합니다. 예를 살펴보겠습니다.

접미사 증가는 1만큼 증가하기 전의 값을 반환합니다.

```kotlin
var a = 10
val b = a++
println(a)  // a = 11
println(b)  // b = 10
```

먼저, 변수의 값을 `a`변수에 할당한 `b`후, 변수의 값을 `a`1만큼 증가시킵니다. 이렇게 하면 `a`는 11이고 `b`는 10입니다.

마찬가지로, 접미사 감소는 다음을 반환합니다.

```kotlin
var a = 10
val b = a--
println(a)  // a = 9
println(b)  // b = 10
```

## 우선순위

일부 작업은 다른 작업보다 우선합니다. 즉, 먼저 수행됩니다. 우선순위가 낮은 순서대로 작업 목록을 살펴보세요.

1. 괄호 ( (expr) );
    
2. 접미사 증가/감소 ( expr++, expr--);
    
3. 단항 더하기/빼기, 접두사 증가/감소 ( -expr, ++expr, --expr );
    
4. 곱셈, 나눗셈 및 모듈러스( *, /, % );
    
5. 덧셈과 뺄셈 ( + , - );
    
6. 할당 연산( =, +=, -=, *=, /=, %= ).
    

산술 표현식 집합을 실행할 때 연산의 우선순위를 고려해야 합니다.

```kotlin
val a = 2
var b = 3
val c = a + 4 * --b  
println(c)   // this is 10
```

감소는 곱셈이나 덧셈보다 우선순위가 높으므로 `--b`먼저 계산됩니다. 산술 연산과 마찬가지로 괄호를 사용하여 연산의 우선순위를 높일 수 있습니다. 또한, 명확성을 위해 괄호를 사용할 수도 있습니다.

```kotlin
var a = 5
val b = 9
val c = 3
val d = a++ + (b / 2) - c - 4
println(d)   // this is 2
```

## 결론

코드를 더 짧고 명확하게 만들려면 할당 연산과 증가/감소 연산을 사용하는 것이 좋습니다. 접두사와 접미사 형식에 대해 다음 사항을 기억하세요. 접두사는 변수를 사용하기 전에 변수를 변경하고, 접미사는 변수를 사용한 후에 변수를 변경합니다. 이제 코틀린에서 이러한 연산을 사용하는 방법을 이해하셨기를 바랍니다.
