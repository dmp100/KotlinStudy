이 수업에서는 **"Hello, World!"**를 출력하는 첫 번째 Kotlin 프로그램을 작성하게 됩니다. 이것은 모든 초보자가 밟아야 할 첫 번째 단계입니다. 프로그램 자체는 매우 간단하지만, 실제로 작동하는 프로그램이며 프로그래밍 언어의 기본 구문을 보여줍니다.



# Hello, World

여기 있습니다. 아래는 이 프로그램의 소스 코드입니다:

```kotlin
fun main() {
    println("Hello, World!")
}
```

코드 설명 컴퓨터에 프로그래밍 환경을 이미 설치했다면, 거기서 프로그램을 실행할 수 있습니다. 그렇지 않다면 걱정하지 마세요. 나중에 다시 설명하겠습니다.

# 기본 용어

이제, 기본 용어를 배우고 우리의 프로그램을 이해해 봅시다.

- **프로그램**은 예측 가능한 방식으로 하나씩 실행되는 일련의 명령어(이를 **문장**이라고 함)입니다. 순차적 흐름은 문장이 작성된 순서대로, 즉 위에서 아래로, 하나씩 실행되는 가장 일반적이고 직관적인 상황입니다;

- **문장**(또는 **프로그래밍 문장**)은 실행될 단일 명령입니다(텍스트 출력과 같은);

**Kotlin에서 문장은 텍스트 출력이나 변수에 값을 할당하는 것과 같은 실행될 단일 명령입니다.** 표현식과 달리, 문장은 계산 결과로 단일 값을 생성하지 않고, 프로그램의 상태를 변경하는 것과 같은 부수 효과를 가진 독립적인 명령입니다. 문장의 결과는 프로그램에서 사용되지 않지만, 표현식의 결과는 종종 사용됩니다. 예를 들어, `val x = 2 * 2` 문장에서 `2 * 2` 표현식이 평가되고, 그 결과가 변수 `x`에 할당됩니다. 하지만 문장 자체의 결과는 프로그램에서 사용되지 않습니다. 대조적으로, `println(2 * 2)`를 작성하면, `2 * 2` 표현식이 평가되고, 그 결과가 직접 `println` 함수에 전달되어 결과를 출력합니다. 모든 문장이 표현식일 수 있지만, 모든 표현식이 문장은 아니라는 점이 중요합니다. 예를 들어, `2 * 2`는 표현식이지만, 프로그램의 상태를 변경하지 않기 때문에 문장은 아닙니다.

- **표현식**은 단일 값을 생성하는 코드 조각입니다(예를 들어, 2*2는 표현식입니다);

**Kotlin에서 표현식은 값으로 평가되는 코드 조각입니다.** 단순한 리터럴 값, 변수, 함수 호출 또는 연산자나 다른 언어 구조를 사용한 이러한 요소들의 더 복잡한 조합일 수 있습니다. 표현식은 함수에서 값을 계산하고 반환하거나, 변수에 값을 할당하거나, 조건문에서 계산이나 연산을 수행하는 데 사용됩니다. 예를 들어, 2 * 2는 표현식이며, 단일 값을 생성합니다.

- **블록**은 **중괄호** `{...}`쌍으로 둘러싸인 0개 이상의 문장 그룹입니다; 우리의 프로그램은 단일 블록으로 구성되어 있습니다;

**Kotlin에서 중괄호** `{}`**는 0개 이상의 문장을 포함할 수 있는 코드 블록을 나타내는 데 사용됩니다.** 또한 문자열 템플릿에서 복잡한 표현식이나 객체 속성을 평가하는 데도 사용됩니다. 표현식이 더 복잡한 경우, 달러 기호 `$` 다음에 오는 표현식 주위에 중괄호를 사용할 수 있습니다. 실수를 방지하고 코드 가독성을 향상시키기 위해 문자열 템플릿의 표현식에 항상 중괄호를 사용하는 것이 좋은 관행입니다.

- **키워드**는 프로그래밍 언어에서 특별한 의미를 가지며 변경할 수 없는 단어입니다;

**Kotlin에서 키워드는 언어에서 특별한 의미를 가지며 프로그래머가 변경할 수 없는 단어입니다.** 코드의 구조와 동작을 정의하는 데 사용됩니다. Kotlin의 키워드 예로는 `fun`(함수 정의에 사용), `var`(변수 정의에 사용), `if`(조건문 정의에 사용) 등이 있습니다. 키워드는 Kotlin에서 예약어이며 식별자로 사용할 수 없습니다.

- **식별자**(또는 **이름**)는 프로그래머가 무언가를 식별하기 위해 작성한 단어입니다;

**Kotlin에서 식별자는 변수, 함수, 클래스 또는 코드의 다른 엔티티에 부여된 이름입니다.** 식별자는 코드를 더 읽기 쉽고 이해하기 쉽게 만드는 데 사용됩니다. 식별자는 그 범위 내에서 고유해야 하며 Kotlin의 명명 규칙을 따라야 합니다. 식별자는 문자, 숫자, 밑줄로 구성될 수 있지만, 숫자로 시작할 수 없습니다. Kotlin에서 예약어인 키워드는 식별자로 사용할 수 없습니다. 식별자는 코드의 다른 요소를 구별하고 프로그램의 기능을 이해하기 쉽게 만드는 데 도움이 됩니다.

- **주석**은 프로그램을 실행할 때 무시되는 텍스트 조각으로, 코드의 일부를 설명합니다. 주석은 `//`로 시작합니다;
- **공백**은 빈 영역, 탭 또는 줄바꿈입니다; 프로그램에서 단어를 구분하고 가독성을 향상시키는 데 사용됩니다.


# 현미경으로 본 "Hello World" 프로그램

**"Hello World"** 프로그램은 모든 Kotlin 프로그램의 기본 요소 사용을 보여줍니다. 지금은 가장 중요한 것들만 살펴보겠습니다.

**진입점.** `fun` 키워드는 실행할 코드 조각을 포함하는 함수를 정의합니다. 이 함수는 특별한 이름인 `main`을 가지고 있습니다. 이는 Kotlin 프로그램의 진입점을 나타냅니다. 함수는 중괄호 `{...}`로 둘러싸인 본문을 가지고 있습니다.

```kotlin
fun main() { 
    // ...
}
```

## 코드 설명

나중에 함수에 대해 논의할 것입니다. 이 함수의 이름은 항상 같아야 합니다: `main`. 만약 `Main`이나 `MAIN` 또는 다른 이름으로 지정하면, 프로그램이 시작되지 않을 것입니다.

**참고:** `//` 뒤의 텍스트는 코드의 일부가 아닌 단순한 주석입니다. 앞으로 주석을 작성하는 방법을 배울 것입니다.

**"Hello, World!" 출력하기.** 이 함수의 본문은 프로그램이 수행할 작업을 정의하는 프로그래밍 문장으로 구성됩니다. 우리의 프로그램은 다음 문장을 사용하여 문자열 **"Hello, World!"**를 출력합니다:

```java
println("Hello, World!")
```

## 코드 설명

이것은 **Hello World** 프로그램에 대해 이해해야 할 가장 중요한 것 중 하나입니다. 우리는 화면에 문자열과 그 뒤에 새 줄을 표시하기 위해 `println` 함수를 호출합니다. 무언가를 출력하기 위해 이 방법을 자주 사용할 것입니다.

**기억하세요** `"Hello, World!"`는 키워드나 이름이 아니라, 화면에 출력될 문자열 리터럴일 뿐입니다.

**Kotlin에서 진입점은 프로그램 실행의 시작점 역할을 하는 특별한 함수입니다.** 이 함수는 'main'이라고 이름 지어져 있으며 항상 같은 이름을 가져야 합니다. 다른 이름으로 지정하면 프로그램이 시작되지 않습니다. 'main' 함수는 중괄호로 둘러싸인 본문을 포함하고 있으며, 그 안에 프로그래밍 문장들이 프로그램이 수행해야 할 작업을 정의합니다. 예를 들어, 'main' 함수는 콘솔에 hello, world!와 같은 문자열을 출력할 수 있습니다. 이것은 Kotlin 프로그램의 진입점이며 Kotlin으로 프로그래밍하는 데 있어 기본적인 개념입니다.


# 여러 문장을 포함하는 프로그램

일반적으로, 프로그램은 여러 문장을 포함합니다. 각 문장을 작성하려면 새로운 줄을 시작해야 합니다. 예를 들어, 아래 프로그램은 두 개의 문장을 가지고 있습니다:

```kotlin
fun main() {
    println("Hello")
    println("World")
}
```

## 코드 설명

이 프로그램을 실행하면, 다음과 같이 표시됩니다:

```kotlin
Hello
World
```

# 결론

축하합니다! 우리는 **"Hello, World!"**를 출력하는 첫 번째 프로그램을 작성했습니다. 이 프로그램은 `main`이라는 하나의 함수를 가지고 있으며, 이는 프로그램의 진입점을 나타냅니다. 지금은 모든 용어(**구문**, **문장**, **블록**)에 대해 걱정하지 마세요 - 이 과정을 통해 설명할 것입니다. 앞으로의 프로그램을 위한 템플릿으로 첫 번째 코드를 사용하는 것을 잊지 마세요.

**Kotlin에서 구문(syntax)은 언어로 프로그램을 작성하는 방법을 규정하는 규칙과 관례의 집합을 의미합니다.** 여기에는 문장, 표현식, 키워드의 구조뿐만 아니라 들여쓰기, 공백, 주석의 사용도 포함됩니다. 예를 들어, 표현식은 리터럴 값, 변수, 함수 호출 또는 연산자를 사용한 이러한 요소들의 조합과 같이 단일 값으로 평가되는 구문의 한 유형입니다. Kotlin의 구문을 이해하는 것은 명확하고, 유지보수 가능하며, 오류 없는 코드를 작성하는 데 필수적입니다.

# 식별자(Identifier)

왼쪽과 오른쪽 항목 연결하기:

- keyword (키워드) - is a word that has a special meaning in the programming language (프로그래밍 언어에서 특별한 의미를 갖는 단어)
    
- name (이름) - is a word written by a programmer that identifies an entity in a program (프로그래머가 프로그램의 엔티티를 식별하기 위해 작성한 단어)
    
- statement (문장) - is a single action (단일 동작)
    
- comment (주석) - is a special text ignored by Kotlin (Kotlin이 무시하는 특별한 텍스트)

# **Find the keyword**

```kotlin
fun main() {
    println("Hello, World!")
}
```

## 코드 설명

이 코드 스니펫(code snippet) 에서 키워드로 분류되는 부분은 무엇인가요?

## 정답

**fun**

키워드는 프로그래밍 언어에서 특별한 의미를 갖는 예약어입니다. 이 코드에서 `fun`은 함수를 정의하는 Kotlin 키워드입니다. 반면에:

- `main`은 함수의 이름(식별자)입니다.
- `()`는 함수의 매개변수 목록을 나타내는 기호입니다.
- `println`은 표준 라이브러리 함수의 이름입니다.
- `"Hello, World!"`는 문자열 리터럴입니다.
