# 기본 리터럴 이해하기

복잡성에 관계없이 모든 프로그램은 기본적으로 숫자, 문자열 및 기타 값에 대한 연산을 수행합니다.

이러한 값들은 **리터럴**이라고 불립니다. 즉, 기호의 가장 기본적인 의미나 의미에서의 값입니다.

첫 번째 프로그램을 작성하기 전에, 

Kotlin의 기본 리터럴인 **정수**, **문자**, **문자열**에 대해 알아봅시다. 이러한 리터럴은 일상 생활 어디에서나 만날 수 있습니다.

리터럴은 코드에서 직접 표현되는 고정된 값으로, 프로그래밍에서 가장 기본적인 데이터 표현 방식입니다.

Kotlin에서 이러한 기본 리터럴들은 다양한 데이터 유형의 기초를 형성하며, 더 복잡한 프로그램을 구축하기 위한 기본 구성 요소가 됩니다.

정수, 문자, 문자열 외에도 Kotlin은 실수(소수점이 있는 숫자), 부울(참/거짓 값), 배열 등 다양한 리터럴 유형을 지원합니다. 

이러한 기본 요소들을 이해하는 것은 효과적인 Kotlin 프로그래밍의 첫 걸음입니다.

# 정수 숫자

실제 세계에서 우리는 사물을 세기 위해 정수를 사용합니다. Kotlin에서도 정수 숫자를 자주 사용하게 될 것입니다.

다음은 쉼표로 구분된 유효한 정수 리터럴의 몇 가지 예입니다: `0`, `1`, `2`, `10`, `11`, `100`.

정수 값에 많은 자릿수가 포함된 경우, 숫자를 더 읽기 쉽게 만들기 위해 밑줄을 추가하여 자릿수를 블록으로 나눌 수 있습니다. 

예를 들어, `1_000_000`은 `1000000`보다 훨씬 읽기 쉽습니다.

원하는 만큼 많은 **밑줄**을 추가할 수 있습니다: `1__000_000`, `1_2_3`. 

단, 밑줄은 숫자의 시작이나 끝에 나타날 수 없다는 것을 기억하세요. `_10` 또는 `100_`과 같이 작성하면 오류가 발생합니다.

# 문자

하나의 문자는 숫자, 글자, 또는 다른 기호를 나타낼 수 있습니다.


단일 문자를 작성하기 위해서는 다음과 같이 기호를 **작은따옴표**로 감싸야 합니다: `'A'`, `'B'`, `'C'`, `'x'`, `'y'`, `'z'`, `'0'`, `'1'`, `'2'`, `'9'`. **문자 리터럴**은 알파벳 글자, `'0'`부터 `'9'`까지의 숫자, 공백(`' '`), 또는 일부 다른 기호(예: `'$'`)를 나타낼 수 있습니다.

숫자를 나타내는 문자(예: `'9'`)와 숫자 자체(예: `9`)를 혼동하지 마세요.

문자는 단일 기호를 나타내기 때문에 두 개 이상의 숫자나 글자를 포함할 수 없습니다. 다음 두 예시는 **잘못된** 것입니다: `'abc'`, `'543'`, 이 리터럴들은 너무 많은 문자를 가지고 있기 때문입니다.

# 문자열

문자열은 광고 텍스트, 웹 페이지 주소 또는 웹사이트 로그인과 같은 텍스트 정보를 나타냅니다. 

문자열은 개별 문자들의 시퀀스입니다.

문자열을 작성하기 위해서는 작은따옴표 대신 **큰따옴표**로 문자들을 감싸야 합니다. 다음은 유효한 예시들입니다: 
`"text"`, `"I want to learn Kotlin"`, `"123456"`, `"e-mail@gmail.com"`. 

따라서, 문자열은 글자, 숫자, 공백 및 기타 문자들을 포함할 수 있습니다.

문자열은 `"A"`와 같이 단 하나의 문자만 포함할 수도 있습니다. 

이것을 문자 `'A'`와 혼동하지 마세요. `'A'`는 문자열이 아닙니다.

# 결론

일상적인 코딩에서 여러분은 리터럴에 크게 의존합니다.

종종 세거나 측정할 때 `123`과 같은 **정수**를, 단일 기호를 위해 `'A'` 또는 `'1'`과 같은 **문자**를, 그리고 텍스트를 처리하기 위해 `"ABC"`와 같은 **문자열**을 사용하게 될 것입니다. 

프로그래밍에서는 효율성이 핵심입니다. 

메모리 사용이 중요한 경우, `"A"`와 같은 문자열 대신 `'A'`와 같은 문자를 선택하세요. 

문자는 일반적으로 더 적은 메모리를 사용합니다.

이것은 단지 시작일 뿐입니다. 

Kotlin은 앞으로의 주제에서 탐색할 수 있는 다양한 다른 리터럴을 제공합니다. 

그럼, 이 세 가지 기본적인 것들을 숙달하기 위해 연습해 봅시다!

# **Address in a city**

**문제**: 도시의 주소를 표현하기 위해 어떤 리터럴이 적합한가요?

**선택지**:

- [x] a number literal (숫자 리터럴)
- [ ] a character literal (문자 리터럴)
- [ ] a string literal (문자열 리터럴)

- 정답 : 숫자 리터럴
도시의 주소를 표현하기 위해 적합한 리터럴은 **string literal(문자열 리터럴)** 입니다.

숫자 리터럴(number literal)은 정답이 아닙니다. 주소는 일반적으로 숫자와 문자가 함께 포함된 텍스트이므로 문자열로 표현하는 것이 적합합니다.

예를 들면:

```kotlin
val address = "서울특별시 강남구 테헤란로 123번길 45"
```

주소는 숫자만으로 표현할 수 없고(숫자 리터럴), 한 글자만으로도 표현할 수 없기 때문에(문자 리터럴) 여러 문자로 구성된 문자열 리터럴이 가장 적합합니다.

# **Select the correct number literal**

**문제**: 올바르게 작성된 숫자 리터럴을 선택하세요.

**선택지**:

- [x] 100_000_000
- [ ] "500"
- [ ] '8'
- [ ] "800_000"

- 정답 100_000_000

맞습니다! 100_000_000이 정확한 숫자 리터럴입니다.

코틀린에서 숫자 리터럴의 올바른 형식은:

- 따옴표 없이 작성됩니다

따라서 100_000_000은 1억을 나타내는 올바른 숫자 리터럴입니다.

나머지 선택지들은 다음과 같은 이유로 올바른 숫자 리터럴이 아닙니다:

- "500": 큰따옴표로 둘러싸여 있어 문자열 리터럴입니다
- '8': 작은따옴표로 둘러싸여 있어 문자 리터럴입니다
- "800_000": 큰따옴표로 둘러싸여 있어 문자열 리터럴입니다


# **Double and single quotes**

**문제**: 빈칸에 알맞은 답을 선택하세요. 

A character should be wrapped in ________ quotes, whereas a string should be wrapped in ________ quotes

**선택지**:

- [ ]  double, double
- [ ]  single, single
- [ ]  double, single
- [x]  single, double

**설명**: 코틀린에서는 문자(Character)는 Char 타입으로 표현되며 작은따옴표(single quotes)로 감싸야 합니다: '1' [Kotlin Help](https://kotlinlang.org/docs/characters.html). 반면에 문자열(String)은 일반적으로 큰따옴표(double quotes)로 감싸진 문자의 시퀀스입니다 [Kotlin Help](https://kotlinlang.org/docs/strings.html). 코틀린에서는 작은따옴표로 문자열을 감쌀 수 없으며, 작은따옴표로 감싸는 것은 오직 하나의 문자일 때만 가능합니다 [Linuxtut](https://linuxtut.com/en/92da2f6a91b8aa59c6ef/). 따라서 문자열 리터럴을 형성하는 데 작은따옴표를 사용할 수 없습니다 [Kotlin](https://kotlinlang.org/docs/tutorials/kotlin-for-py/strings.html).



# **String to characters**

**문제**: 문자열 리터럴 "Kot"를 문자 리터럴로 나누려고 합니다. 올바른 결과는 무엇인가요?

**선택지**:

- [x]  'K', 'o', 't'
- [ ]  'K', 'o', "t"
- [ ]  "K", "o", "t"
- [ ]  K, o, t
- [ ]  "Ko", 't'

**설명**: 코틀린에서 문자열(String)은 큰따옴표(double quotes)로 표현되고, 개별 문자(Character)는 작은따옴표(single quotes)로 표현됩니다. 따라서 문자열 "Kot"를 개별 문자로 나누면 각 문자는 작은따옴표로 표현되어야 합니다: 'K', 'o', 't'. 다른 선택지들은 큰따옴표를 사용하거나 따옴표가 없어 올바르지 않습니다.


# **Statements**

**문제**: 리터럴에 관한 올바른 설명을 선택하세요.

**선택지**:

- [ ]  **"123"은 올바른 숫자 리터럴이다
- [x]  **"abc"는 올바른 문자열 리터럴이다
- [ ]  **'ab'는 올바른 문자 리터럴이다

**해설**:

- **"123"**은 올바른 숫자 리터럴이 아닙니다. 숫자 리터럴은 따옴표 없이 작성해야 합니다(예: 123). 큰따옴표로 감싸진 "123"은 문자열 리터럴입니다.
- **"abc"**는 올바른 문자열 리터럴입니다. 코틀린에서 문자열은 큰따옴표(" ")로 감싸야 합니다.
- **'ab'**는 올바른 문자 리터럴이 아닙니다. 코틀린에서 문자(Character) 리터럴은 작은따옴표(' ')로 감싸야 하지만, 단 하나의 문자만 포함할 수 있습니다. 예를 들어 'a'는 올바른 문자 리터럴이지만, 'ab'는 두 개의 문자를 포함하므로 올바르지 않습니다.

# **Correct string writing**

**문제**: 올바르게 작성된 **문자열(string)** 리터럴을 선택하세요.

**선택지** (하나 이상 선택 가능):

- [ ] 443
- [x] "q"
- [x] "2312"
- [ ] 'q'
- [ ] '5'
- [x] "text"

**참고**:

- "Report a typo", "See hint", "Get unstuck" 버튼은 실제 문제 환경의 인터페이스 요소입니다.

**해설**:

- **443**: 이것은 따옴표 없이 작성된 숫자 리터럴이므로 문자열 리터럴이 아닙니다.
- **"q"**: 큰따옴표로 감싸져 있어 올바른 문자열 리터럴입니다.
- **"2312"**: 큰따옴표로 감싸져 있어 올바른 문자열 리터럴입니다. 숫자들이지만 큰따옴표로 감싸져 있으므로 문자열로 취급됩니다.
- **'q'**: 작은따옴표로 감싸져 있어 문자(Character) 리터럴이지, 문자열 리터럴이 아닙니다.
- **'5'**: 작은따옴표로 감싸져 있어 문자(Character) 리터럴이지, 문자열 리터럴이 아닙니다.
- **"text"**: 큰따옴표로 감싸져 있어 올바른 문자열 리터럴입니다.

코틀린에서 문자열(String) 리터럴은 항상 큰따옴표(" ")로 감싸야 합니다. 작은따옴표(' ')는 문자(Character) 리터럴에만 사용됩니다. 따옴표 없이 작성된 숫자는 숫자 리터럴입니다.


# **An incorrect literal**

**문제**: 잘못된 리터럴을 찾으세요.

**선택지**:

- [ ] "123456"
- [ ] "e-mail@gmail.com"
- [ ] 1806
- [ ] '@'
- [x] 'abc'

**참고**:

- "Report a typo", "See hint", "Get unstuck" 버튼은 실제 문제 환경의 인터페이스 요소입니다.

**해설**:

- **"123456"**: 큰따옴표로 감싸진 올바른 문자열 리터럴입니다.
- **"e-mail@gmail.com"**: 큰따옴표로 감싸진 올바른 문자열 리터럴입니다.
- **1806**: 따옴표 없이 작성된 올바른 숫자 리터럴입니다.
- **'@'**: 작은따옴표로 감싸진 올바른 문자(Character) 리터럴입니다.
- **'abc'**: 잘못된 리터럴입니다. 코틀린에서 문자(Character) 리터럴은 작은따옴표(' ')로 감싸야 하지만, 작은따옴표 안에는 단 하나의 문자만 포함할 수 있습니다. 'abc'는 세 개의 문자를 포함하고 있으므로 올바르지 않습니다.

코틀린에서:

- 문자열(String)은 큰따옴표(" ")로 감싸야 합니다.
- 문자(Character)는 작은따옴표(' ')로 감싸야 하며, 단 하나의 문자만 포함할 수 있습니다.
- 숫자 리터럴은 따옴표 없이 작성합니다.


# **An invalid literal**

**문제**: 유효하지 않은 리터럴을 선택하세요.

**선택지**:

- [x] '10000000'
- [ ] "10000000"
- [ ] 10000000
- [ ] 10_000_000

**참고**:

- "Report a typo", "See hint", "Get unstuck" 버튼은 실제 문제 환경의 인터페이스 요소입니다.

**해설**:

- **'10000000'**: 유효하지 않은 리터럴입니다. 코틀린에서 문자(Character) 리터럴은 작은따옴표(' ')로 감싸야 하지만, 작은따옴표 안에는 단 하나의 문자만 포함할 수 있습니다. '10000000'은 여러 개의 문자를 포함하고 있으므로 올바르지 않습니다.
- **"10000000"**: 큰따옴표로 감싸진 올바른 문자열(String) 리터럴입니다.
- **10000000**: 따옴표 없이 작성된 올바른 정수 리터럴입니다.
- **10_000_000**: 코틀린에서는 숫자 리터럴에 가독성을 위해 언더스코어(_)를 사용할 수 있습니다. 이것은 올바른 정수 리터럴입니다.

코틀린에서:

- 문자(Character)는 작은따옴표(' ')로 감싸며, 단 하나의 문자만 포함할 수 있습니다.
- 문자열(String)은 큰따옴표(" ")로 감싸야 합니다.
- 숫자 리터럴은 따옴표 없이 작성하며, 가독성을 위해 언더스코어(_)를 사용할 수 있습니다.
