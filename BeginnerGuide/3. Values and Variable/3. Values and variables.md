# 변수(variable)란 무엇인가?

변수(variable)는 문자열(string), 숫자(number), 또는 다른 것이 될 수 있는 값(value)을 저장하는 저장소입니다. 모든 변수(variable)는 다른 변수와 구별하기 위한 **이름(name)**(또는 **식별자(identifier)**)를 가집니다. 변수의 이름을 통해 값에 접근할 수 있습니다. 변수(variable)는 프로그램에서 가장 자주 사용되는 요소 중 하나이므로 사용 방법을 이해하는 것이 중요합니다.

## 변수 선언하기(Declaring variables)

변수(variable)를 사용하기 전에 먼저 선언해야 합니다. 변수를 선언하기 위해 Kotlin은 두 가지 **키워드(keywords)**를 제공합니다:

- `val`(_value_의 약자)은 **읽기 전용 변수(read-only variable)**(단지 **명명된 값(named value)**)를 선언하며, **초기화(initialized)** 후에는 변경할 수 없으므로 값은 한 번만 할당될 수 있습니다(사실 이것은 완전히 사실이 아니며, 나중에 이 문제에 대해 더 자세히 논의할 것입니다);
- `var`(_variable_의 약자)은 변경 가능한 **가변 변수(mutable variable)를 선언하며, (필요한 만큼 여러 번) 변경할 수 있습니다.
- `const`는 **컴파일 시간에 알려진(known at compile-time)** 변수(val과 함께 사용)에 사용됩니다.

**Kotlin에서 키워드(keyword)는 언어에서 특별한 의미를 가지며 프로그래머가 변경할 수 없는 단어입니다.** 키워드는 코드의 구조와 동작을 정의하는 데 사용됩니다. Kotlin의 키워드 예로는 `fun`(함수를 정의하는 데 사용), `var`(변수를 정의하는 데 사용), `if`(조건문을 정의하는 데 사용) 등이 있습니다. 키워드는 Kotlin에서 예약어이며 식별자로 사용할 수 없습니다.

`val`과 `var` 키워드 모두 변수(variable)를 제공합니다!

변수를 선언할 때, 이 두 키워드 중 하나 뒤에 변수 이름을 추가해야 합니다. 주의할 점: 변수 이름은 숫자로 시작할 수 없습니다. 일반적으로 문자로 시작합니다. 코드를 쉽게 이해할 수 있도록 변수에 의미 있고 읽기 쉬운 이름을 선택해야 합니다. 변수에 특정 값을 할당하려면 할당 연산자(assignment operator) `=`를 사용해야 합니다. 문자열 `"Kotlin"`으로 초기화된 **불변 변수(immutable variable)** `language`를 선언해 봅시다.

**Kotlin에서 초기화(initialization)는 변수가 선언될 때 초기 값을 할당하는 것을 의미합니다.** 이것은 값 뒤에 `=` 기호를 사용하여 수행할 수 있습니다. `val` 키워드를 사용할 때는 초기화가 필요합니다. 기본 클래스에 생성자 매개변수가 있는 경우, 파생 클래스는 이를 처리해야 합니다. Kotlin은 기본 클래스가 제대로 초기화되지 않으면 프로그램이 컴파일되도록 허용하지 않습니다. 초기화는 간단한 객체에 대해서는 가벼운 작업이지만, 더 복잡한 클래스의 인스턴스를 생성하는 것은 비용이 많이 들 수 있습니다.

```kotlin
val language = "Kotlin"
```

이제 변수 이름으로 이 문자열에 접근할 수 있습니다. 출력해 봅시다!

```kotlin
println(language) // 따옴표 없이 "Kotlin"을 출력합니다
```

이 변수는 `val`로 선언되었기 때문에 초기화 후에는 수정할 수 없습니다.

**Kotlin에서 불변 변수(immutable variable)는 선언되고 초기화된 후에는 변경하거나 수정할 수 없는 변수입니다.** 이는 `val` 키워드를 사용하여 선언되며, 변수에 새 값을 재할당할 수 없습니다. 이는 데이터의 무결성을 보장하고 의도하지 않은 수정을 방지하는 데 도움이 됩니다. 또한 불변 변수는 코드를 더 읽기 쉽고 이해하기 쉽게 만들 수 있습니다. 가능한 한 불변 변수를 사용하고 절대적으로 필요한 경우에만 가변 변수를 사용하는 것이 좋은 방법입니다.

이름은 대소문자를 구분합니다: `language`는 `Language`와 동일하지 않습니다.

이제 `dayOfWeek`라는 가변 변수(mutable variable)를 선언하고 변경 전후에 그 값을 출력해 봅시다.

```kotlin
var dayOfWeek = "Monday"
println(dayOfWeek) // Monday를 출력합니다

dayOfWeek = "Tuesday"
println(dayOfWeek) // Tuesday를 출력합니다
```

위의 예에서, 우리는 `dayOfWeek`라는 변수를 선언하고 `"Monday"`라는 값으로 초기화했습니다. 그런 다음 변수 이름으로 값에 접근하여 출력했습니다. 그 후, 변수의 값을 `"Tuesday"`로 변경하고 이 새로운 값을 출력했습니다.

값을 변경하기 위해 변수를 다시 선언할 필요는 없습니다. 그냥 `=` 연산자를 사용하여 새 값을 할당하면 됩니다.

다른 변수의 값으로 변수를 선언하고 초기화하는 것도 가능합니다:

```kotlin
val cost = 3
val costOfCoffee = cost
println(costOfCoffee) // 3을 출력합니다
```

## 다양한 유형의 값 저장하기(Storing different types of values)

이미 변수가 다양한 유형의 값을 저장할 수 있다고 언급했습니다: 문자열(strings), 숫자(numbers), 문자(characters), 그리고 나중에 만나게 될 다른 **데이터 타입(data types)**.

**Kotlin에서 데이터 타입(data type)은 변수, 표현식 또는 함수 매개변수가 보유할 수 있는 값의 종류를 지정하는 분류입니다.** 이는 변수와 함수 매개변수에 대한 명시적 또는 암시적 타입 선언이 필요한 Kotlin과 같은 정적 타입 언어에서 필수적인 측면입니다. 예를 들어, Kotlin에는 정수, 부동 소수점 숫자, 문자, 불리언, 문자열과 같은 다양한 데이터 타입이 있습니다. 이 중 문자열은 따옴표로 둘러싸인 0개 이상의 문자 시퀀스를 나타내는 널리 사용되는 데이터 타입입니다. 문자열은 종종, 조건문과 정규표현식에서 사용되며, 길이를 가져오거나 두 문자열을 연결하는 등의 다양한 작업을 가지고 있습니다. 더불어, 타입 추론(type inference)은 Kotlin의 유용한 기능으로, 컴파일러가 변수, 표현식 또는 함수 매개변수의 데이터 타입을 자동으로 추론할 수 있게 하여 코드를 더 간결하고 읽기 쉽게 만듭니다.

숫자, 문자열, 문자를 저장하는 세 개의 불변 변수(immutable variables)를 선언하고 값을 출력해 봅시다.

```kotlin
val ten = 10
val greeting = "Hello"
val firstLetter = 'A'

println(ten) // 10을 출력합니다
println(greeting) // Hello를 출력합니다
println(firstLetter) // A를 출력합니다
```

가변 변수(mutable variables)(키워드 `var`로 선언된 것)에는 한 가지 제한이 있습니다. 값을 재할당할 때, 초기 값과 동일한 타입의 새 값만 사용할 수 있습니다. 따라서 아래의 코드는 올바르지 않습니다:

```kotlin
var number = 10
number = 11 // 괜찮습니다
number = "twelve" // 여기서 오류가 발생합니다!
```

이 제한을 기억해 주세요!

## 결론(Conclusion)

이제 변수를 선언하는 데 사용되는 두 가지 키워드가 있다는 것을 알게 되었습니다. 실제로, 많은 경우에 불변 변수(키워드 `val`로 선언된 것)를 사용하는 것이 더 좋습니다. `var`를 사용하기 전에 그 경우에 `val`이 적합하지 않은지 확인해야 합니다. 정말로 적합하지 않다면 `var`를 사용하세요. 하지만, 코드에 가변 변수가 많을수록 읽고 이해하기가 더 어렵다는 점을 명심하세요. 불변 변수는 더 읽기 쉬운 코드를 작성하는 데 도움이 됩니다. 그러니, 가능한 한 `val`을 사용하세요! 절대적으로 필요한 경우에는 쉽게 `var`로 대체할 수 있습니다.

따라서 정답은 2, 3, 4번 옵션입니다.

# **할당 연산자(The assignment operator)**

Kotlin에서 할당 연산자는 다음과 같이 생겼습니다. (The assignment operator in Kotlin looks like)

- [ ] ->
- [ ] :=
- [x] =
- [ ] <-

## 해설:

Kotlin에서 할당 연산자는 `=` 기호입니다. 이 연산자를 사용하여 변수에 값을 할당합니다.

예시:

```kotlin
val language = "Kotlin"
var number = 10
number = 20 // 값 재할당
```

다른 옵션들은 Kotlin의 할당 연산자가 아닙니다. `->`는 주로 람다 표현식에서 사용되고, `:=`와 `<-`는 Kotlin에서 할당 연산자로 사용되지 않습니다.

# **아주 큰 숫자(Very Big Number)**

Kotlin에서 `bigNumber`라는 변수를 선언하고 정수값 `100_000_000`(여기서는 밑줄이 허용됨)으로 초기화하세요. 다른 것은 출력하지 마세요!

```kotlin
fun main() {
    // do not touch the lines above
    // declare and initialize a variable here
    var bigNumber = 100_000_000
    // do not touch the lines below
    println(bigNumber)
}
```

## 해설:

이 코드는 `bigNumber`라는 변수를 선언하고 값 `100_000_000`으로 초기화합니다.

Kotlin에서는 큰 숫자를 더 읽기 쉽게 만들기 위해 숫자 사이에 밑줄(`_`)을 사용할 수 있습니다. 이는 코드의 가독성을 높이는 데 도움이 되며, 컴파일러는 이러한 밑줄을 무시합니다.

여기서는 `var` 키워드를 사용하여 가변 변수를 선언했지만, 값이 변경되지 않으므로 `val` 키워드를 사용해도 똑같이 작동했을 것입니다.


# **선언 수정하기(Fix declarations)**

두 변수 `a`와 `b`의 잘못된 선언이 주어졌습니다. 이 선언들을 수정하세요. 지금은 코드 스타일 오류는 무시하세요.

```kotlin
fun main() {    
    // Fix declaration. Do not touch the lines above
    val a = 512343
    val b = 3431231
    // Do not touch the lines below
    println("$a $b")
}
```

## 해설:

원래 코드에서 변수 `a`와 `b`의 선언이 잘못되었을 것입니다. 위 코드에서는 각 변수에 대해 적절한 `val` 키워드를 사용하여 정수값으로 초기화했습니다.

Kotlin에서 변수를 선언할 때는 `val`(불변 변수) 또는 `var`(가변 변수) 키워드를 사용해야 합니다. 여기서는 값이 변경되지 않으므로 `val`을 사용하는 것이 적절합니다.

또한 변수 이름과 할당 연산자(`=`) 사이, 그리고 할당 연산자와 값 사이에 올바른 공백을 추가했습니다.


# **프로그램 수정하기(Fix the program)**

`string` 값을 출력해야 하는 프로그램이 주어졌지만 작동하지 않습니다. 이를 수정해 주세요.

**예시 출력(Sample Output)**

```
Hello, Kotlin!
```

```kotlin
fun main() {
    var string = "Hello, World!"
    string = "Hello, Kotlin!"
    println(string)
}
```

## 해설:

이 코드는 이미 올바르게 작동합니다. 수정할 필요가 없습니다.

코드는 다음과 같이 작동합니다:

1. `var` 키워드를 사용하여 `string`이라는 가변 변수를 선언하고 "Hello, World!"로 초기화합니다.
2. 그 다음 줄에서 `string` 변수의 값을 "Hello, Kotlin!"으로 변경합니다.
3. 마지막으로 `println` 함수를 사용하여 `string` 변수의 현재 값인 "Hello, Kotlin!"을 출력합니다.

`var` 키워드로 선언된 변수는 값을 변경할 수 있기 때문에 이 코드는 예상대로 작동합니다. 만약 `val` 키워드를 사용했다면, 변수 값을 변경하려고 할 때 오류가 발생했을 것입니다.


# **변수의 이름(The name of a variable)**

다음과 같은 변수 선언이 주어졌습니다:

```kotlin
val speed = "high"
```

이 변수의 이름은 무엇인가요?

- [ ] val
- [ ] "high"
- [x] speed

## 해설:

- `=`은 할당 연산자입니다.

따라서 이 변수의 이름은 `speed`입니다.


# **val 또는 var?(val or var?)**

`number`라는 변수가 있습니다. 이 값을 5로 변경하세요. 최소한 두 가지 방법이 있다는 점에 유의하세요.

```kotlin
fun main() {    
    // Do not touch the line above
    var number = 2
    number = 5
    // Do not touch the lines below
    print(number)
}
```

## 해설:

이 코드는 `number`라는 변수를 선언하고 초기값 2를 할당한 다음, 그 값을 5로 변경합니다.

변수 값을 변경하는 두 가지 방법이 있습니다:

1. 위 코드처럼 `var` 키워드로 변수를 선언하고 나중에 값을 변경하는 방법:
    
    ```kotlin
    var number = 2
    number = 5
    ```
    
2. 이미 원하는 값으로 변수를 초기화하는 방법:
    
    ```kotlin
    val number = 5
    ```
    

첫 번째 방법에서는 `var` 키워드를 사용해야 합니다. 왜냐하면 `var`로 선언된 변수만 값을 변경할 수 있기 때문입니다. `val`로 선언된 변수는 초기화 후에 값을 변경할 수 없습니다.

두 번째 방법에서는 `val` 키워드를 사용할 수 있습니다. 이 경우에는 변수 값을 변경할 필요가 없으므로 불변 변수를 사용하는 것이 더 좋은 방법일 수 있습니다.


# **불변 변수 선언하기(Declaring an immutable variable)**

읽기 전용 변수의 선언을 시작하는 키워드는 무엇인가요?

- [ ] var
- [x] val
- [ ] final
- [ ] let

## 해설:

Kotlin에서 읽기 전용(불변) 변수를 선언할 때는 `val` 키워드를 사용합니다. `val`은 "value"의 약자로, 한 번 초기화되면 그 값을 변경할 수 없습니다.

```kotlin
val language = "Kotlin" // 읽기 전용 변수 선언
```

다른 옵션들에 대한 설명:

- `var`: 가변 변수를 선언할 때 사용하는 키워드입니다. 값을 변경할 수 있습니다.
- `final`: Java에서 사용되는 키워드로, Kotlin에서는 이에 해당하는 `val`을 사용합니다.
- `let`: Kotlin에서 변수 선언 키워드가 아니라 스코프 함수 중 하나입니다.


# 할당(Assignments)

네 개의 변수가 주어졌습니다. 이 변수들에 값을 할당하세요. 변수 `one`은 1, `two`는 2, `three`는 3, `four`는 4여야 합니다. 코드 품질은 신경 쓰지 말고 작업을 완료하세요. 곧 이를 최적화하는 방법을 알게 될 것입니다.

```kotlin
fun main() {
    val one = 1
    val two = 2
    val three = 3
    val four = 4
    println("$one $two $three $four")
}
```

## 해설:

Kotlin에서 `val` 키워드로 선언된 변수는 선언과 동시에 초기화되어야 합니다. 위 코드에서는 각 변수에 적절한 값을 할당했습니다.

원래 코드에서 `four`가 400으로 초기화되어 있었지만, 문제 요구사항에 따라 4로 변경했습니다.

각 변수가 요구사항대로 초기화되었습니다:

- `one` = 1
- `two` = 2
- `three` = 3
- `four` = 4

이 값들은 마지막 줄에서 문자열 템플릿(`$variable`)을 사용하여 출력됩니다.

# **선언(Declarations)**

올바른 변수 선언을 모두 선택하세요.

- [ ] name = "John"
- [x] var x = 10
- [ ] let cost = 500
- [ ] val = 5
- [x] val greeting = "Hi"

## 해설:

올바른 변수 선언은 다음과 같습니다:

- **var x = 10**: 올바른 선언입니다. `var` 키워드를 사용하여 가변 변수 `x`를 선언하고 값 10으로 초기화했습니다.
    
- **val greeting = "Hi"**: 올바른 선언입니다. `val` 키워드를 사용하여 불변 변수 `greeting`을 선언하고 문자열 "Hi"로 초기화했습니다.
    

잘못된 선언은 다음과 같습니다:

- **name = "John"**: 잘못된 선언입니다. Kotlin에서 변수를 선언할 때는 `var` 또는 `val` 키워드가 필요합니다.
    
- **let cost = 500**: 잘못된 선언입니다. `let`은 Kotlin에서 변수 선언 키워드가 아니라 스코프 함수입니다.
    
- **val = 5**: 잘못된 선언입니다. 변수 이름이 없습니다. 올바른 형식은 `val variableName = 5`입니다.
    

따라서 정답은 2번과 5번 옵션입니다.


# **가변 변수 선언하기(Declaring a mutable variable)**

가변 변수를 선언하는 데 사용되는 키워드는 무엇인가요?

- [ ] now
- [ ] let
- [ ] val
- [ ] no keyword needed (키워드 필요 없음)
- [x] var

## 해설:

Kotlin에서 가변 변수(값을 변경할 수 있는 변수)를 선언할 때는 `var` 키워드를 사용합니다. `var`은 "variable"의 약자입니다.

```kotlin
var dayOfWeek = "Monday"
dayOfWeek = "Tuesday" // 값 변경 가능
```

다른 옵션들에 대한 설명:

- `now`: Kotlin에서 변수 선언 키워드가 아닙니다.
- `let`: Kotlin에서 변수 선언 키워드가 아니라 스코프 함수 중 하나입니다.
- `val`: 읽기 전용(불변) 변수를 선언할 때 사용되는 키워드입니다. 초기화 후에는 값을 변경할 수 없습니다.
- `no keyword needed`: 잘못된 옵션입니다. Kotlin에서는 변수를 선언할 때 `var` 또는 `val` 키워드가 반드시 필요합니다.

# **교육 앱에서 변수 할당하기(Assigning variables in an educational app)**

학습 플랫폼을 위한 애플리케이션을 개발하고 있다고 가정해 보세요. 메인 함수에서 학습자의 진행 상황을 추적하는 일수와 어떤 프로그래밍 언어를 공부하고 있는지 나타내는 언어, 두 개의 변수를 할당해야 합니다. 애플리케이션은 사용자에게 배우고 있는 언어를 보여주고 1씩 증가한 현재 학습 일수를 표시하는 인사말을 표시해야 합니다. 코드에는 채워야 할 세 개의 빈칸이 있습니다. 이 빈칸에 무엇을 넣을 것인가요?

```kotlin
fun main() {
    var count = 10
    val language = "Kotlin"
    println("Hello, World! You are learning $language")
    count += 1
    println("This is your $count day of learning")
}
```

## 해설:

코드의 빈칸을 다음과 같이 채웠습니다:

1. `var count = 10`: 학습자의 진행 일수를 추적하는 가변 변수 `count`를 선언하고 10으로 초기화했습니다. 이 값은 나중에 변경되어야 하므로 `var` 키워드를 사용했습니다.
    
2. `val language = "Kotlin"`: 배우고 있는 프로그래밍 언어를 나타내는 불변 변수 `language`를 선언하고 "Kotlin"으로 초기화했습니다. 이 값은 변경되지 않으므로 `val` 키워드를 사용했습니다.
    
3. `count += 1`: 학습 일수를 1 증가시킵니다. 이는 `count = count + 1`과 동일합니다.
    

출력 결과는 다음과 같을 것입니다:

```
Hello, World! You are learning Kotlin
This is your 11 day of learning
```

이 코드는 Kotlin 문자열 템플릿(`$variable`)을 사용하여 변수 값을 문자열에 삽입하는 방법을 보여줍니다.


# **변수에 관한 올바른 설명(Correct description about variables)**

변수에 관한 올바른 서술을 모두 선택하세요.

- [x] 변수는 값을 저장할 수 있습니다(A variable can store a value)
- [ ] 변수의 이름은 변경될 수 있습니다(The name of a variable can be changed)
- [ ] **var** 키워드로 선언된 변수의 값은 한 번만 변경될 수 있습니다(The value of a variable declared with the keyword **var** can be changed only once)
- [x] 변수는 이름(식별자)을 가집니다(A variable has a name (an identifier))

## 해설:

1. **변수는 값을 저장할 수 있습니다** - 올바른 설명입니다. 변수의 기본 목적은 값을 저장하는 것입니다.
    
2. **변수의 이름은 변경될 수 있습니다** - 잘못된 설명입니다. 변수가 한 번 선언되면, 그 이름은 변경할 수 없습니다. 새로운 이름이 필요하다면 새 변수를 선언해야 합니다.
    
3. **var 키워드로 선언된 변수의 값은 한 번만 변경될 수 있습니다** - 잘못된 설명입니다. `var` 키워드로 선언된 변수는 여러 번 값을 변경할 수 있습니다. 한 번만 값을 할당할 수 있는 것은 `val` 키워드로 선언된 변수입니다.
    
4. **변수는 이름(식별자)을 가집니다** - 올바른 설명입니다. 모든 변수는 다른 변수와 구별하기 위한 고유한 이름(식별자)을 가집니다.
    

따라서 정답은 1번과 4번 옵션입니다.
