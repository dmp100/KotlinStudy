## 명명이 중요한 이유

아시다시피 모든 변수는 고유한 이름을 가져야 합니다. 변수 이름을 짓는 것은 쉬워 보일 수 있습니다. 하지만 사실은 그렇지 않습니다. 숙련된 프로그래머는 코드를 쉽게 이해할 수 있도록 변수 이름을 신중하게 선택합니다. 프로그래머는 다른 사람이 작성한 코드를 읽는 데 많은 시간을 쏟기 때문에 변수 이름은 매우 중요합니다. 변수 이름을 잘못 지으면 코드를 읽을 수 없게 될 것입니다.

이런 상황을 어떻게 방지할 수 있을까요? 모든 변수에 항상 설명적이고 간결한 이름을 붙이세요. 그래야 동료들이 여러분의 코드를 즐겁게 읽을 수 있을 것입니다. 변수 이름을 제한하는 **두 가지 규칙이 있습니다.**

## 명명 규칙

Kotlin에는 여러 가지 규칙이 있습니다.

- 이름은 대소문자를 구분합니다( `number`와 같지 않음 `Number`).
    
- 각 이름에는 문자, 숫자, 밑줄만 포함될 수 있습니다.
    
- 이름은 숫자로 시작할 수 없습니다.
    
- 이름은 키워드가 될 수 없습니다(예: `val`, `var`, `fun`는 허용되지 않습니다).
    

따라서 변수 이름에는 공백이 허용되지 않습니다. 공백은 백틱과 함께만 사용할 수 있습니다.

```kotlin
val `good name` = 5
val bad name = 2 // will not work
```

이러한 규칙을 어기면 프로그램이 작동하지 않습니다.

유효한 변수 이름을 살펴보세요.

```kotlin
score, level, fruitType, i, j, abc, _cost, number1, `name with space`
```

아래는 잘못된 내용입니다.

```kotlin
@pple, $dollar, 1number, !ab, val, var, _, name with space
```

## 변수 명명 규칙

Kotlin은 또한 다음과 같은 규칙을 제공합니다.

- 변수 이름이 한 단어인 경우 소문자로 입력합니다(예: — `number`, `value`).
    
- 변수 이름에 여러 단어가 포함된 경우 각 단어를 .으로 묶어 `lowerCamelCase`첫 번째 단어는 소문자로, 나머지 단어는 대문자로 시작합니다(예: — `numberOfCoins`).
    
- [변수를 밑줄](https://hyperskill.org/learn/step/4389 "코틀린에서 밑줄은 단어 경계, 특히 영숫자 문자 또는 밑줄과 영숫자가 아닌 문자 사이의 경계를 나타내는 약어입니다. | 특정 문자와 일치하지 않고, 영숫자 또는 밑줄과 공백 문자, 문자열의 시작 또는 끝과 같은 영숫자가 아닌 문자 사이의 경계와 일치합니다. 특정 문자로 시작하거나 끝나는 모든 단어, 또는 영숫자가 아닌 문자가 앞뒤에 오는 문자를 일치시키는 데 사용할 수 있습니다.") 로 시작하지 마세요 `_`. 하지만 기술적으로는 가능합니다.
    
- 변수에 의미 있는 이름을 선택하세요. 예를 들어, 둘 다 유효하지만, `score`보다 의미가 더 있는 이름을 선택하세요.`s`
    

## 마법의 숫자

이제 상수의 이름과 사용법에 대해 알아보겠습니다. 코드에서 [상수를](https://hyperskill.org/learn/step/4389 "코틀린에서 상수는 `val` 키워드로 선언되는 속성 유형이며, 할당 후에는 수정할 수 없습니다. | 상수 값은 컴파일 시점에 결정되며 기본 유형의 값으로 초기화해야 합니다. 상수 값은 screaming\_snake\_case 방식으로 명명해야 합니다. 상수 변수는 프로그램 실행 중에 여러 번 접근할 수 있지만 값은 변경되지 않습니다. 또한 코틀린은 컴파일 시점에 상수 값을 선언하기 위한 `const` 키워드를 제공하며, 이는 최상위 상수에 사용할 수 있습니다. 상수를 사용하면 오류를 방지하고 코드의 가독성을 높일 수 있습니다.") 사용해야 하는 경우가 있습니다 . 예를 들어, 항상 7인 요일 수를 출력해야 할 때입니다. 하지만 다음 코드는 다른 사람들(그리고 잠시 후 자신에게도)에게 그다지 유익하지 않을 것입니다.

```kotlin
println(7)
```

는 무엇인가요 `7`? 왜 출력되나요? 전혀 모르겠네요. 코드에서 이런 값을 **매직 넘버** 라고 합니다 . 매직 넘버는 꼭 숫자일 필요는 없습니다. 데이터 유형이 아니라 이러한 변수의 "마법 같은" 본질에 대한 것입니다. 아무도 그 변수의 기원, 용도, 의미를 알지 못합니다. 그러니 매직 넘버를 사용하지 마세요.

그러면 우리는 이러한 가치를 어떻게 다루어야 할까?

[먼저, 이러한 값은 변경 불가능한 변수](https://hyperskill.org/learn/step/4389 "코틀린에서 불변 변수는 선언되고 초기화된 후에는 변경하거나 수정할 수 없는 변수입니다. `val` 키워드를 사용하여 선언하며, 이 키워드는 변수에 새 값을 재할당할 수 없도록 합니다. 이는 데이터 무결성을 보장하고 의도치 않은 수정을 방지하는 데 도움이 됩니다. 또한, 불변 변수는 코드의 가독성과 이해도를 높여줍니다. 가능하면 불변 변수를 사용하고, 변경 가능한 변수는 절대적으로 필요한 경우에만 사용하는 것이 가장 좋습니다.") , 즉 상수 에 저장되어야 합니다 . 여기서는 단순히 로 선언된 값만을 말하는 것이 아닙니다 . 상수란 함수 또는 다른 함수 외부에 `val`존재하는 값을 의미합니다 . 이러한 상수는 **완전히** 변경 불가능한 상수입니다(이 설명이 모든 것을 설명하는 것은 아니며, 나중에 자세히 설명하겠습니다). 다음은 그 예입니다.`const val``main()`

```kotlin
const val DAYS_OF_THE_WEEK = 7

fun main() {
    // ...
    println(DAYS_OF_THE_WEEK) // 7
    // ...
}
```

둘째, 상수 이름은 의미가 있어야 하며, 상수 이름을 지을 때는 `SCREAMING_SNAKE_CASE`스타일을 사용해야 합니다.  
  
다음은 상수 이름의 잘못된 예입니다.

```kotlin
const val s = 4
```

컴파일은 되지만, 코드 자체만으로는 설명이 부족합니다. 다음은 좋은 예시입니다.

```kotlin
const val SEASONS = 4
```

상수를 만들고 의미 있는 이름을 지정한 다음 숫자를 해당 상수로 바꿉니다.

## 결론

변수 이름을 짓는 것은 정말 중요한 일이지만, 위의 규칙은 **선택 사항** 입니다 . 따라서 위의 규칙을 따르는 것이 좋습니다. 앞서 언급했듯이, 이러한 규칙은 코드의 가독성을 높이고 동료들의 작업도 원활하게 해 줄 수 있습니다.
