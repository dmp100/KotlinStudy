다양한 예상치 못한 상황에 따라 서로 다른 유형의 예외가 존재합니다. 이러한 예외는 모두 서로 관련되어 있으며, 유형 계층 구조를 형성합니다. 여기에서는 이러한 계층 구조의 핵심 사항을 살펴보고, 발생할 수 있는 가장 일반적인 Kotlin 예외 유형을 검토하며, 이러한 유형들이 서로 어떻게 연결되는지 살펴보겠습니다.

## 하위 유형과 상위 유형

먼저 용어를 알아보겠습니다. 코틀린의 타입은 하위 타입-슈퍼 타입 관계의 계층 구조로 구성됩니다. 하위 타입과 슈퍼 타입은 무엇일까요? 예를 들어 보겠습니다.

커피나 차를 좋아하시나요? 음, 둘 다 음료입니다. 커피와 차는 모두 특정 유형, 즉 '음료'와 관련이 있다고 할 수 있습니다. 코틀린 용어로, 커피와 차는 ' 음료'의 **하위 유형** 이고 , '음료'는 이 두 유형의 **상위 유형 입니다.**

![Kotlin의 유형 및 하위 유형의 예](https://ucarecdn.com/531e1446-070f-4b4a-a376-a566687fbbc3/)

하위 **유형 은 다른 데이터 유형(** _슈퍼 유형_ ) 과 관련되어 있으며, 해당 데이터 유형에서 공통적인 특성과 동작 규칙을 상속받는 데이터 유형입니다 . 하위 _유형_ 마다 동작 규칙이 다를 수 있습니다. 이 예에서 모든 종류의 음료는 어떤 종류의 색깔을 가지고 있지만, 커피와 차는 서로 다른 색깔을 가지고 있습니다.

논리적으로, **슈퍼타입은** 그 속성과 메서드가 하위타입이 따라야 할 특성과 동작 규칙을 지정하는 타입입니다.

## 예외 계층 개요

다음 그림은 예외의 단순화된 계층 구조를 보여줍니다.

![단순화된 예외 계층](https://ucarecdn.com/f7ffede8-355a-4d41-b140-61e470f02a33/)

  
보시다시피, 이 계층 구조의 기반은 타입입니다 `Throwable`. 이 타입은 계층 구조의 루트이며, 모든 예외는 어떤 식으로든 이 타입에 속합니다. 이 타입은 예외 처리를 위한 유용한 메서드 집합을 제공합니다. 그리고 코틀린의 모든 예외는 타입 의 자손 이므로 `Throwable`, 이러한 모든 메서드는 각 예외에서 사용할 수 있습니다. 다음 항목에서 이에 대해 자세히 살펴보겠습니다.

이 유형 `Throwable`에는 두 가지 직접적인 하위 유형이 있습니다. `Error`와 입니다 `Exception`. 는 `Error`일반적인 애플리케이션에서는 처리해서는 안 되는 심각한 문제를 나타냅니다. 프로그램 코드의 오류 처리에 대해서는 생각하지 않아야 합니다. 여기서는 에 초점을 맞출 것입니다. `Exception`애플리케이션을 개발할 때 일반적으로 예외를 포착하고 처리하게 됩니다. 하위 유형과 상위 유형은 상대적인 용어이므로 주어진 유형이 둘 중 하나일 필요는 없습니다. 이 경우 는 의 `Exception`하위 유형 `Throwable`이자 의 상위 유형이기도 합니다 `RuntimeException`.

및 와 `Exception`같은 유형 의 하위 유형은 애플리케이션 내부의 예외적인 이벤트를 처리합니다. 오늘은 의 종류에 대해 더 자세히 살펴보겠습니다 .`RuntimeException``IOException``RuntimeException`

## 산술 예외

`ArithmeticException`[는 코드가 잘못된 산술 연산을](https://hyperskill.org/learn/step/13264 "Kotlin에서 산술 연산은 특정 연산자를 사용하여 숫자 값에 대해 수행되는 수학적 계산입니다. Kotlin에는 5가지 산술 연산자가 있습니다. 더하기(+), 빼기(-), 곱하기(\*), 나누기(/), 모듈로(%). 이러한 연산자는 이진 연산자로, 연산자가 적용되는 값이나 변수인 두 개의 피연산자를 사용합니다. 예를 들어, 표현식 1 + 3에서 1과 3은 피연산자이고 +는 연산자입니다. Kotlin의 산술 연산은 곱셈이 덧셈이나 뺄셈보다 우선순위가 높은 것과 같은 특정 규칙을 따릅니다. 괄호를 사용하여 실행 순서를 지정하고 표기법을 단순화할 수 있습니다. 또한 Kotlin은 산술 연산과 할당을 결합하는 복합 할당 연산을 지원합니다. 예를 들어 덧셈 후 할당(a += b) 및 뺄셈 후 할당(a -= b)이 있습니다.") 시도할 때 발생하는 런타임 예외입니다 . 매우 일반적인 예를 살펴보겠습니다.

```kotlin
val example = 2 / 0 // throws ArithmeticException
```

0으로 나누는 것은 허용되지 않으므로, 이 줄은 오류를 발생시키고 `ArithmeticException`프로그램 실행이 중단됩니다.

## 숫자형식예외

계산 중에 또 다른 유형의 예외가 발생할 수 있습니다. `NumberFormatException`이 예외는 메서드가 숫자를 입력받을 것으로 예상하지만 실제 입력값이 다른 경우 발생합니다. 예를 들어 보겠습니다.

```kotlin
val string = "It's not a number"
val number = string.toInt() // throws NumberFormatException
```

여기서 문자열을 `"It's not a number"`숫자로 변환해 보려고 했지만 `Int`, 이는 당연히 불가능하므로 a가 `NumberFormatException`반환되었습니다.

## IndexOutOfBoundsException

`IndexOutOfBoundsException`존재하지 않는 인덱스에 접근할 때 발생합니다. 예를 들어, 요소가 5개뿐인 컬렉션의 10번째 요소에 접근하려고 하면 인덱스가 범위를 벗어나게 됩니다.

이 유형의 예외에는 더 많은 하위 유형이 있습니다. 다음 예제에서는 (예상하셨겠지만) 문자열에 사용되는 `StringIndexOutOfBoundsException`특수 유형 인 를 throw합니다.`IndexOutOfBoundsException`

```kotlin
val sequence = "string"
println(sequence[10])   // throws StringIndexOutOfBoundsException
```

기억하시겠지만, 문자열은 각 문자에 인덱스가 할당된 문자들의 집합으로 표현될 수 있습니다. 문자열에 지정된 인덱스에 해당하는 문자가 없으면 a가 `StringIndexOutOfBoundsException`throw됩니다.

## 결론

Kotlin의 모든 예외는 단일 계층 구조에 포함됩니다. 모든 예외는 <type>에 속하며 `Exception`, <type>은 <type>의 하위 유형입니다 `Throwable`.

의 중요한 상속 유형 중 하나는 `Exception`다음과 같습니다 `RuntimeException`. 이 유형에는 프로그램 실행 중 발생할 수 있는 예외가 포함됩니다. 하위 유형으로는 `ArithmeticException`, `NumberFormatException`, , 가 있습니다 `IndexOutOfBoundsException`.
