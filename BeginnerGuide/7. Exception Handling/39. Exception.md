다른 많은 언어와 마찬가지로 코틀린에서도 예상치 못한 상황이 발생할 수 있습니다. 프로그램을 작성하고 실행하려고 한다고 상상해 보세요. 갑자기 코틀린 컴파일러가 코드가 올바르지 않다고 표시하고 오류를 표시합니다. 하지만 어떤 컴파일러도 만능이 아니므로 코틀린 컴파일러가 모든 오류를 방지할 수는 없습니다. 따라서 프로그램이 구문적으로 올바르고 아무 문제 없이 컴파일되었더라도 실행 중에 오류가 발생할 수 있습니다. 이러한 상황을 코틀린에서는 **예외** 라고 하며 , 이 개념에 대해 살펴보겠습니다.

## 예외 발생

먼저, 예외에 대해 설명해 보겠습니다. 예외는 예상치 못한 비표준적인 이벤트로, 다양한 상황에서 발생하며 프로그래머에게 불편을 줄 수 있습니다. 아래 예를 살펴보겠습니다.

```kotlin
fun readNextInt(): Int {
    return readln().toInt()
}

fun runIncrementer() {
    val increment = 1 + readNextInt()
    println(increment)
}

fun main() {
    runIncrementer()
}
```

방금 표준 입력에서 값을 읽어오는 함수를 만들었습니다. 아주 간단한 예시입니다. 한 줄을 읽어 숫자로 변환한 후 결과를 반환합니다. 혹시 문제가 있을까요? 자, 여기 있습니다. 사용자가 다음과 같은 내용을 입력하지 않을 거라고 확신할 수 없습니다.

```kotlin
> Hi :)
```

아차! 프로그램을 실행하고 이 줄을 입력하면 예외가 발생하고 프로그램이 종료되면서 다음과 같은 출력이 나옵니다.

```no-highlight
Exception in thread "main" java.lang.NumberFormatException: For input string: "Hi :)"
	at java.base/java.lang.NumberFormatException.forInputString(NumberFormatException.java:67)
	at java.base/java.lang.Integer.parseInt(Integer.java:661)
	at java.base/java.lang.Integer.parseInt(Integer.java:777)
	at org.example.MainKt.readNextInt(Main.kt:4)
	at org.example.MainKt.runIncrementer(Main.kt:8)
	at org.example.MainKt.main(Main.kt:13)
	at org.example.MainKt.main(Main.kt)
```

`NumberFormatException`즉, 4번째 줄의 함수에서 a를 얻는다는 뜻입니다 `readNextInt()`.

## 예외란 무엇인가

보시다시피, 코드에 문제가 있어도 프로그램 실행은 중단되지 않지만, 예외라는 오류가 있는 줄을 실행하려고 하면 프로그램이 충돌합니다 **.** 예외는 프로그램 실행(런타임) 중에 발생하는 오류이고, 구문 오류는 프로그램 컴파일 중에 발생하는 오류입니다. 예외는 프로그램의 정상적인 실행을 방해합니다.

예외는 잘못된 산술 연산을 수행하거나 위 예처럼 잘못된 입력을 받는 등 다양한 이유로 발생할 수 있습니다. 이러한 오류는 예상치 못한 동작이 발생했음을 나타내며, 입력 데이터의 유효성을 검사하거나 처리 방식을 변경해야 합니다. 다른 예를 살펴보고 예외를 방지하는 방법을 알아보겠습니다.

## 0으로 나누기

예를 들어, 간단한 일이 있다고 가정해 보겠습니다. 우유를 샀는데, 얼마를 썼는지, 그리고 몇 팩이나 가지고 있는지 알고 있습니다. 각 팩의 가격을 알고 싶으시다면, 아주 간단합니다.

```kotlin
fun itemPrice(total: Int, amount: Int): Int {
    return total / amount
}
```

`amount`하지만 equal to를 사용하여 이 함수를 호출하면 어떻게 될까요 `0`? 아시다시피 0으로 나눌 수는 없습니다. 다음과 같은 결과가 나옵니다.

```kotlin
Exception in thread "main" java.lang.ArithmeticException: / by zero
	at org.example.MainKt.itemPrice(Main.kt:4)
	at org.example.MainKt.main(Main.kt:8)
	at org.example.MainKt.main(Main.kt)	at org.example.MainKt.itemPrice(Main.kt:4)
```

`ArithmeticException`즉, 4번째 줄의 함수에서 을 얻는다는 뜻입니다 `itemPrice()`.

음, 또 다른 예외가 발생했습니다. 이 경우에는 다음과 같이 하면 오류를 아주 간단하게 방지할 수 있습니다.

```kotlin
fun itemPrice(total: Int, amount: Int): Int {
    if (amount == 0) {
        println("Division by zero")
        return 0
    }
    return total / amount
}
```

추가적인 검증을 통해 예외를 방지할 수 있는 경우가 많으므로 예외 상황에 대해서도 생각해 두는 것을 잊지 마세요.

## 결론

요약해 보겠습니다. 예외는 프로그램의 컴파일 및 실행을 막지는 않지만, 예외를 유발하는 줄을 실행하려고 하면 프로그램이 충돌합니다. 제어문은 프로그램에서 일부 예외를 방지하는 데 도움이 될 수 있습니다. 나중에 예외를 [처리](https://hyperskill.org/learn/step/7243 "코틀린에서 예외 처리란 프로그램 실행 중 예외를 처리하고 대응하는 과정을 의미합니다. | 이 작업은 `try` 및 `catch` 키워드를 사용하여 수행됩니다. `try` 블록에는 예외를 발생시킬 수 있는 코드가 포함되어 있고, `catch` 블록에는 예외 발생 시 수행할 작업이 포함되어 있습니다. `try` 블록에서 예외가 발생하면 런타임 시스템은 처음부터 시작하여 해당 예외를 처리할 첫 번째 적합한 `catch` 블록을 결정합니다. 예외를 기반으로 올바른 결정을 내릴 수 있는 충분한 정보가 있는 메서드에서 예외를 처리하는 것이 가장 좋습니다.") 하고 직접 예외를 throw하는 일반적인 방법을 배우게 될 것입니다. 작업 잘 되길 바랍니다!
