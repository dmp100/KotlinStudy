오류가 발생하는 경우를 살펴보고 이를 방지하는 방법을 알아보겠습니다.

## 컴파일 타임 오류

컴파일 타임 오류는 프로그램 컴파일을 방해하는 오류입니다. 예를 들면 다음과 같습니다.

- 구문 오류: 잘못된 키워드, 오타, 괄호 또는 중괄호 누락
    
- 잘못된 가져온 패키지 이름
    
- 존재하지 않는 메서드 호출.
    

다음 컴파일 타임 오류 예를 살펴보겠습니다. 다음 프로그램은 **"Hello!"** 라는 문자열을 출력해야 하지만 컴파일되지 않습니다.

```kotlin
fun main(args: Aray<String>) {
    printn("Hello!")
}
```

이 프로그램에는 두 가지 오류가 있습니다.

- 클래스 이름의 오타 `Array`;
    
- 함수의 이름이 잘못되었습니다 `println`.
    

이러한 실수를 수정하면 프로그램을 컴파일할 수 있습니다.

[이러한 오류를 방지하기 위해 프로그래머는 정적 코드 분석기가](https://hyperskill.org/learn/step/11559 "Kotlin에서 정적 코드 분석기는 프로그램 컴파일 전에 컴파일 타임 오류를 발견하는 데 도움이 되는 도구입니다. | 컴파일 타임 오류는 일반적으로 프로그램 실행을 방해하기 때문에 식별 및 수정이 더 쉽습니다. 이러한 오류에는 오타와 잘못된 메서드 호출이 포함됩니다. 통합 개발 환경(IDE)의 정적 코드 분석기는 프로그래머가 프로그램 컴파일을 시도하기 전에 컴파일 타임 오류를 발견하고 수정하는 데 도움을 줄 수 있습니다. 이를 통해 디버깅 과정에서 시간과 노력을 절약할 수 있습니다. 또한, 정적 코드 분석기를 갖춘 최신 IDE는 코드의 더 복잡한 오류와 취약점에 대한 경고를 강조 표시할 뿐만 아니라 코드 개선 방법에 대한 팁도 제공합니다.") 포함된 최신 IDE(통합 개발 환경)를 사용합니다 . 이 도구를 통해 프로그래머는 컴파일 전에 컴파일 타임 오류를 파악할 수 있습니다. 또한, 코드의 복잡한 오류와 취약한 부분에 대한 경고와 코드 개선 팁을 제공합니다.

시간이 지나면 컴파일 오류가 적거나 아예 없는 코드를 작성하게 될 것입니다.

## 런타임 오류

런타임 오류("버그"라고도 함)는 프로그램 실행 중에 발생하는 오류입니다. 런타임 오류로 인해 프로그램이 예기치 않게 작동하거나 실행이 중단될 수도 있습니다.

런타임 오류에는 두 가지 하위 유형이 있습니다.

- **[논리 오류](https://hyperskill.org/learn/step/11559 "코틀린에서 논리 오류는 코드가 올바르게 작성되지 않아 프로그램이 잘못된 결과를 생성할 때 발생하는 런타임 오류의 한 유형입니다. | 컴파일 타임 오류와 달리 논리 오류는 프로그램 실행을 방해하지 않고, 예상치 못한 동작을 하거나 잘못된 출력을 생성하게 합니다. 논리 오류의 일반적인 예로는 프로그램이 예상대로 hello! 대신 hi!를 출력하는 경우입니다. 논리 오류는 명확한 오류 메시지를 생성하지 않고, 신중한 디버깅, 코드 검토 및 자동 테스트를 통해 식별해야 하므로 감지하고 수정하기 어려울 수 있습니다.")** – 코드가 올바르지 않아 프로그램이 잘못된 결과를 생성하는 경우(예를 들어, **"Hello!"** 대신 프로그램에서 **"Hi!"** 가 출력되는 경우 )
    
- 0으로 나누기, 파일을 찾을 수 없음, 기타 예상치 못한 경우와 같이 **처리되지 않은 예외입니다 .**
    

이후 수업에서는 예외적인 사건(예외)을 처리하는 방법을 알아보겠습니다.

런타임 오류를 피하는 것은 컴파일 오류를 피하는 것보다 더 어려운 작업입니다. 프로그램이 성공적으로 컴파일된다고 해서 버그가 전혀 없다는 보장은 없습니다. 이러한 오류를 찾는 데는 여러 가지 방법이 있습니다.

- 프로그램을 **디버깅** 하려면
    
- 프로그램에 대한 **자동 테스트를** 작성하려면
    
- **[개발 프로세스의 일부로 코드 검토](https://hyperskill.org/learn/step/11559 "코틀린에서 코드 리뷰는 개발자가 작성한 코드를 검토하고 평가하여 품질, 스타일 및 기능 기준을 충족하는지 확인한 후 메인 코드베이스에 병합하는 프로세스입니다. | 이러한 관행은 협업 개발 환경에서 일관성을 유지하고 오류를 포착하며 팀원 간의 지식 공유를 촉진하는 데 필수적입니다. 코틀린에서는 KDoc 및 Dokka와 같은 문서화 도구를 사용하여 코드 리뷰를 용이하게 할 수 있습니다. 이러한 도구는 소스 코드의 주석을 기반으로 API 문서를 생성하여 리뷰어가 코드의 목적과 동작을 더 쉽게 이해할 수 있도록 합니다.")** 관행을 활용하는 것입니다 . 일반적으로 이 관행은 한 명 이상의 개발자가 프로그램의 소스 코드를 시각적으로 검토하는 경우를 의미합니다.
    

## 결론

- 컴파일 타임 오류는 프로그램이 컴파일되지 않을 때 발생합니다. 여기에는 오타나 잘못된 메서드 호출이 포함됩니다.
    
- IDE의 정적 코드 분석기는 프로그램을 컴파일하기 전에 컴파일 타임 오류를 발견하는 데 도움이 됩니다.
    
- 런타임 오류 또는 버그는 프로그램 실행 중 컴파일 후 발생할 수 있습니다. 이로 인해 프로그램이 예기치 않게 동작하거나 충돌이 발생할 수 있습니다.
    
- 디버깅은 프로그램의 런타임 오류를 식별하는 데 유용한 도구입니다.
